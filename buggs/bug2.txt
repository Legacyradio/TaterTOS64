TaterTOS64v3 - fry525 — Harden ELF loader program-header bounds checks
========================================================================

TIMESTAMP
---------
2026-03-04 02:37:48 CST

WHY THIS LOG
------------
- Reported high-severity bug in ELF loader bounds handling.
- Loader trusted `e_phoff/e_phnum/p_offset/p_filesz` from container payload
  without fully validating ranges against `payload_size`.
- Crafted `.fry/.tot` files could drive out-of-bounds reads in kernel context.

BUG
---
- File: `src/kernel/proc/elf.c`
- Paths: around previous load flow at `payload + eh->e_phoff` and
  `src = payload + off` with segment copy loops.
- Impact: kernel OOB read on malformed containers via invalid program header
  table ranges and/or segment file ranges.

FIX APPLIED
-----------
- Added ELF program-header table validation before dereferencing `ph`:
  - require `e_phentsize == sizeof(struct elf64_phdr)`
  - require `e_phoff <= payload_size`
  - require `e_phnum * e_phentsize` to fit within payload from `e_phoff`
- Added per-`PT_LOAD` segment validation before copy/mapping math:
  - require `p_filesz <= p_memsz`
  - require `p_offset + p_filesz` fully within payload (overflow-safe)
  - guard `vaddr + memsz` overflow
  - guard page-alignment addition overflow before calculating mapping end

WHY THIS FIX
------------
- Eliminates attacker-controlled pointer/range traversal past container payload.
- Rejects malformed ELF metadata up front with `ELF_LOAD_ERR_BOUNDS` or
  `ELF_LOAD_ERR_BAD_ELF_HEADER`.

FILES CHANGED
-------------
- src/kernel/proc/elf.c
- Makefile
- tools/elf_loader_bounds_test.c

BUILD / VERIFY COMMANDS
-----------------------
- `PATH="/opt/cross/bin:$PATH" make kernel`
- `make test-elf-bounds`

VERIFY RESULTS
--------------
- `make kernel`: PASS (exit code 0)
- `make test-elf-bounds`: PASS (all malformed corpus cases)

NOTES
-----
- `test-elf-bounds` is an opt-in host-side regression target only.
- Runtime fix in `elf.c` is always active during normal boot/process load.
