Handoff Follow-up - 2026-02-21

Summary
- Fixed SMP trampoline allocation by reserving a fixed low page and using it directly.

Changes
1) src/kernel/mm/pmm.c
   - Reserve page at 0x8000 (SMP trampoline) after freeing usable pages.

2) src/drivers/smp/smp.c
   - Use fixed trampoline address 0x8000 instead of pmm_alloc_page_below().
   - Removes "no low memory for trampoline" failure path.

Why
- AP startup code must live below 1MB. PMM was failing to allocate low memory.
  A fixed, reserved low page avoids allocation failure and matches the plan.
