Handoff Follow-up - 2026-02-21

Summary
- Fixed AP trampoline far jumps to use linear address (base + offset) for PM and LM entries.

Changes
1) src/drivers/smp/trampoline.asm
   - Build far pointers using tramp_base + PM_ENTRY_OFF / LM_ENTRY_OFF.
   - Use `jmp dword far [ptr]` for both PM and LM transitions.
   - Added pm_farptr/lm_farptr data slots.

Why
- Previous far jump used offset only (no base), so AP jumped to 0x0000:PM_ENTRY_OFF instead
  of 0x8000+offset, causing #GP at 0x8008.
