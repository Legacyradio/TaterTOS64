PROBLEM: VMD NVMe not found in ECAM (32 buses scanned)
  Log showed: VMD: MEMBAR1=0X404A10F000 then NO NVME FOUND IN ECAM

ROOT CAUSE: Intel VMD DID=0x1911 BAR layout:
  MEMBAR0 = BAR0+BAR1 (offsets 0x10,0x14) = VMD control registers
  MEMBAR1 = BAR2+BAR3 (offsets 0x18,0x1C) = PCIe ECAM config space  <-- NEEDED
  MEMBAR2 = BAR4+BAR5 (offsets 0x20,0x24) = MSI-X routing table
  Code was reading BAR0+BAR1 (MEMBAR0 = control registers) as the ECAM base.
  The printed "MEMBAR1=0x404A10F000" was actually MEMBAR0.
  The real ECAM (MEMBAR1) is at BAR2+BAR3 - a different physical address.
  Scanning the control-register region found no PCIe devices (all 0xFFFF).

SECONDARY: ECAM region was mapped write-back (no VMM_FLAG_CACHE_DISABLE).
  PCI config space MMIO must be uncacheable.
