=== CHANGES ===

src/user/gui/gui.c (full rewrite):
  - window_t gains: shm_id (-1=no TW), shm_ptr, shm_w/h, done flag,
    msgbuf[256]/msglen byte accumulator, textbuf[1024]/textlen text ring
  - process_window_output(): parses fry_proc_output bytes for TW_MAGIC:
      TW_MSG_CREATE_WINDOW → fry_shm_alloc(w*h*4), fry_shm_map(id),
        resize window chrome to SHM dims, send tw_msg_win_created_t back
        to app via fry_proc_input (app then maps its own copy of the SHM)
      TW_MSG_UPDATE → consumed; blit happens every frame if shm_ptr set
      plain text bytes → appended to textbuf for draw_window_text()
  - draw_window_text(): scrolling multiline text renderer (12px line height)
  - draw_window(): blit shm_ctx via gfx_blit() if shm_ptr set, else text
  - Main loop: keyboard forwarded to focused window via fry_proc_input
  - done flag prevents repeated strcat of "[DONE]" on dead-process polls
  Why: GUI becomes a compositor; TW apps share SHM pixel buffers

src/user/apps/sysinfo.c (full rewrite):
  - TaterWin client: fry_write(1, tw_msg_create_win_t) → request window
  - Waits for tw_msg_win_created_t via fry_read(0,...), up to 3s (300 tries)
  - fry_shm_map(resp.shm_id) → maps shared physical pages into own VA space
  - render(): clears SHM, draws uptime/procs/backlight/battery with gfx_*
  - Loop: render() → fry_write(1, TW_MSG_UPDATE) → fry_sleep(1000)
  - Static fallback_pixels[] used if SHM alloc fails (window stays black)
  Why: first real TaterWin app; live-updating graphical info panel
