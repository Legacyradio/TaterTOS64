WHAT CHANGED (why)
=======================================================================
- `src/kernel/acpi/namespace.c`
  - Updated `ns_lookup()` to resolve `ACPI_NODE_ALIAS` nodes while traversing
    each path segment and again at return.
  - Why: `_REG/_INI` lookups can fail on firmware trees that route names via
    aliases, producing false "invocation failed" even when methods exist.

- `src/kernel/acpi/extended.c`
  - Improved `invoke_path_method()` failure diagnostics:
    - logs explicit lookup failure path
    - logs non-method node type with path
  - Why: separate path-resolution failures from method-type failures in boot
    logs so EC handshake debugging is deterministic.
