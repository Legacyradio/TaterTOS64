fry430 — Fix DMI PID selection for Dell 7530 (A30E) + no-write PCR diag probing
=======================================================================
Date: 2026-03-01 18:10:34 UTC

WHAT CHANGED (why)
=======================================================================
- `src/kernel/acpi/extended.c`
  - Updated `pcr_get_dmi_pid_candidates()` ordering for `LPC 8086:A30E`:
    - Prefer `PID_DMI_CANNONLAKE (0x88)` first
    - Keep `PID_DMI_DEFAULT (0xEF)` as fallback
    - Keep `PID_DMI_PANTHERLAKE (0x2F)` as fallback
  - Why: Dell Precision 7530 is Coffee Lake/Cannon Lake PCH family; primary
    source shows DMI PID is `0x88` for this family.

  - Updated PID tie-break behavior in `ec_mirror_pcr_ioe()`:
    - Tie-break now prefers candidate[0] from runtime candidate ordering
      instead of always preferring `PID_DMI_DEFAULT`.
  - Why: keep PID preference hardware-aware and runtime-detected.

  - Updated no-write diagnostics path (when BIOS IOE bits are already set):
    - Replaced fixed `PID_DMI_DEFAULT` read with runtime candidate probing and
      scoring across all DMI PID candidates.
    - Persist selected diagnostic PID into `g_pcr_pid_used`.
    - Added per-PID diagnostic probe logging.
  - Why: avoid false `0xFFFFFFFF` diagnostics caused by forcing the wrong PID
    in read-only mode.

  - Updated stale comment about wrong/correct PID behavior to generic wording
    that avoids hardcoded platform assumptions.
  - Why: keep code comments accurate and hardware-agnostic.
