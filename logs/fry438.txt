Code changes:

1) src/kernel/acpi/extended.c
   A) Added explicit eSPI status masks and helpers:
      - ESPI_*_STATUS_MASK constants
      - espi_status_mask_for_off()
      - espi_status_bits()
      WHY:
        Raw register values include enable/default bits that can stay set
        by design; those must not be treated as active faults.

   B) Reworked espi_clear_errors():
      - Detects "error present" from masked status bits only.
      - Writes only status bits for W1C clear (not full raw register values).
      - Evaluates g_espi_clear_ok on status bits only.
      - Logs both raw and masked status snapshots per pass/final.
      WHY:
        Prevents false "clear failed" results when only non-status bits remain.

   C) Updated espi_diag_probe() clear + summary logic:
      - Uses status-bit masks for optional clear writes.
      - Uses status-bit masks for warning/impairment messages.
      WHY:
        Keeps diagnostics aligned with actual fault status semantics.

2) src/user/shell/shell.c
   - acpibreak now computes masked status bits for pc/vw/fc/ln.
   - Break-stage verdict now keys on status bits (not raw non-zero).
   - Output now prints both raw and status bit views and includes FC.
   WHY:
     Avoids false eSPI-error verdicts from persistent enable/default bits.

3) src/user/apps/sysinfo.c
   - eSPI error coloring now keys on masked status bits.
   - eSPI error line now shows raw pc/vw/fc/ln values for context.
