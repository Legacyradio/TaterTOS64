SYMPTOM
==========================================================================
Opening Shell triggered a user-mode page fault and fault cascade.
From qemu trace:
  RIP=0x40140b  CR2=ffff80000063f010
  fault site mapped to malloc() append path.

ROOT CAUSE
==========================================================================
User libc syscall wrappers modeled only RCX/R11 as clobbered, but kernel
syscall return path does not preserve all caller-saved GPRs. Optimized user
code (malloc) held allocator state across syscall and that state was corrupted.

FIX
==========================================================================
File:
  /home/jjsako/TaterTOS64v3/src/user/libc/libc.c

Changes:
- Updated syscall wrapper inline asm constraints/clobbers to model actual
  register clobber behavior:
  - syscall3: +D/+S/+d and clobber r8/r9/r10/rcx/r11
  - syscall2: +D/+S and clobber rdx/r8/r9/r10/rcx/r11
  - syscall1: +D and clobber rsi/rdx/r8/r9/r10/rcx/r11
  - syscall0: clobber rdi/rsi/rdx/r8/r9/r10/rcx/r11

WHY
==========================================================================
Prevents compiler from keeping live allocator/process state in registers that
the kernel syscall path may overwrite, eliminating the shell startup crash.
