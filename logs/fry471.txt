CHANGES IMPLEMENTED
-------------------
Files:
- src/kernel/fs/ntfs.c
- src/kernel/fs/ntfs.h

1) Hardware-agnostic byte-granular partition reads
- Added ntfs_dev_read_bytes() in ntfs.c.
- Reads arbitrary byte ranges using bd->sector_size at runtime.
- Removes hidden assumption that NTFS logical sectors == block-device sectors.

2) Boot sector read reworked to tiered geometry validation
- ntfs_read_boot_sector() now:
  - reads first 512 bytes via byte-granular helper,
  - reads BPB bytes/sector (offset 0x0B),
  - validates bytes/sector (power-of-two, 512..4096),
  - reads full boot sector of that size,
  - validates 0x55AA at BPB sector tail.

3) NTFS LBA reads now honor NTFS sector units explicitly
- ntfs_read_sectors() now treats rel_lba/count in NTFS bytes_per_sector units
  and translates to byte offsets before device read.
- Prevents sector-size mismatch math from silently reading wrong ranges.

4) MFT runlist robustness
- Increased NTFS_MAX_RUNS from 128 -> 512 (ntfs.h).
- ntfs_decode_data_runs() now rejects malformed/truncated runlists instead of
  silently returning partial data.
- Added negative-LCN and zero-run guards.

5) Mount-time MFT coverage validation
- ntfs_mount() now computes total bytes covered by decoded $MFT runs.
- Mount fails if run coverage is smaller than $MFT real_size.
- This prevents partial mounts that can list names but fail stat/open later.
