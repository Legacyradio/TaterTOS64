========================================================================

TIMESTAMP
---------
2026-03-03 04:30:57 UTC

USER-REPORTED SYMPTOMS
----------------------
1) File Manager showed:
   - empty list
   - `readdir failed: /nvme`
   even though NVMe was detected and mount entries appeared.
2) Clicking FAT32 NVMe entry (ex: `/nvme1`) could hard-freeze UI.

ROOT CAUSE SUMMARY
------------------
1) Secondary mount selection could assign `/nvme` to a candidate that mounted
   structurally but failed root directory access at runtime.
2) File Manager performed `stat()` across many entries immediately after
   `readdir()`, which can stall interaction on slow/problematic directories.

CHANGES IMPLEMENTED
-------------------
1) Kernel secondary mount root-probe gate
   File: src/kernel/fs/vfs.c
   Why:
   - Prevent `/nvme` from pointing at a mount that cannot serve root dir
     `stat/readdir`, which directly causes `readdir failed: /nvme`.
   What:
   - Added `sec_probe_mount_root()`:
     - validates mount root as directory via `ops->stat(fs_data, "", &st)`
     - validates root enumeration via `ops->readdir(fs_data, "", ...)`
   - Applied probe before publishing secondary mounts for:
     - ToTFS
     - NTFS
     - FAT32
   - If probe fails, candidate is rejected and not mounted at `/nvme`/`/nvmeN`.

2) File Manager stat-budget and lazy fallback
   File: src/user/apps/fileman.c
   Why:
   - Avoid synchronous stat storms causing visible hangs during navigation.
   What:
   - Added reload budget:
     - max 96 `stat()` calls OR 250ms per directory reload.
   - Beyond budget, mark entries unresolved instead of forcing full probe pass.
   - Defer expensive directory probe (`readdir(full)`) until user opens
     unresolved entry.
   - Preserves navigation while reducing freeze risk on FAT32 NVMe paths.
