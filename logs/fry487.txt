WHY THESE CHANGES
-----------------
- NTFS device reads were issuing one-sector NVMe commands for every small
  fragment, which can explode into thousands of synchronous commands when
  listing a directory and trigger a system hang on hardware.
- Batch aligned reads into multi-sector requests to reduce NVMe command
  volume and shorten worst-case directory enumeration hangs.

CODE CHANGES
------------
1) src/kernel/fs/ntfs.c
   - Reworked `ntfs_dev_read_bytes` to:
     - Handle unaligned head/tail with single-sector reads.
     - Issue bulk aligned reads up to 1MB per request.
     - Avoid repeated per-sector NVMe commands for contiguous ranges.
