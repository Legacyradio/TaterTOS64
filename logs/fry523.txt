WHY THIS LOG
------------
- User requested finding and fixing a bug in TaterTOS64v3.
- A concrete correctness bug was found in shell `vfsdbg` directory parsing.

BUG
---
- File: `src/user/shell/shell.c`
- Function: `vfs_readdir_brief`
- Issue: dirent records from `fry_readdir_ex` were parsed without validating
  full record bounds (`off + rec_len <= n`) and name payload bounds
  (`name_len + 1 <= rec_len - sizeof(struct fry_dirent)`).
- Risk: malformed/truncated directory records could cause out-of-bounds reads
  while printing debug listing.

FIX APPLIED
-----------
- Added strict bounds checks before reading variable-length dirent name:
  - break if `off + rec_len > n`
  - compute `payload = rec_len - sizeof(struct fry_dirent)`
  - skip record when `name_len + 1 > payload`
- This preserves existing behavior while preventing unsafe record access.

- src/user/shell/shell.c
