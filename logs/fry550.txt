CONTEXT
-------
- Follow-up review noted `nvme_pick_nsid()` used a hardcoded loop bound:
  `for (uint32_t i = 0; i < 1024; i++)`.
- The namespace list buffer is page-sized (4096 bytes), so 1024 entries is
  correct today but should be derived from buffer size rather than magic
  numbers.

WHY
---
- Keeps bounds tied to declared capacity and avoids hidden assumptions.
- Improves readability/maintainability with explicit intent.

CHANGE APPLIED
--------------
- File updated: `src/drivers/storage/nvme.c`
  - Added:
    - `#define NVME_PAGE_BYTES 4096u`
    - `#define NVME_NS_LIST_ENTRIES (NVME_PAGE_BYTES / (uint32_t)sizeof(uint32_t))`
  - Updated `nvme_pick_nsid()` loop bound:
    - From: `i < 1024`
    - To:   `i < NVME_NS_LIST_ENTRIES`

EFFECT
------
- NSID scan count now derives from buffer byte size and element width.
- Behavior remains unchanged (4096 / 4 == 1024 entries).
