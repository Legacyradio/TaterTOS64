CHANGES APPLIED
---------------
- Files updated:
  - `src/drivers/smp/smp.h`
    - Declared `smp_bsp_index()`.
  - `src/drivers/smp/smp.c`
    - Added `smp_bsp_index()` helper to return the actual BSP logical index.
  - `src/drivers/timer/lapic_timer.c`
    - Changed BSP timer gating from hardcoded logical CPU 0 to the actual BSP.
    - Why: MADT discovery order does not guarantee the BSP is stored at index 0.
      On real hardware this could let the first scheduler tick/runqueue advance
      on the wrong logical CPU assumption.
  - `src/kernel/proc/sched.c`
    - Changed BSP-only runqueue routing and BSP-only syscall stack bookkeeping
      from hardcoded logical CPU 0 to `smp_bsp_index()`.
    - Why: runnable boot userspace tasks and BSP scheduler state must track the
      actual bootstrap processor, not a guessed logical slot.

- The first LAPIC timer tick should drive the intended BSP scheduler queue.
