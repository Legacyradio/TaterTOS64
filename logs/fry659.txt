  entries whose first dword is still the separator value:

    - `0xAAAABBBB` for paging boundary

  SEC_RT stream, CPU2 found no remaining loadable runtime sections, and
  `iwl_upload_runtime_cpu_image()` returned `-11`, surfacing as:
    - `FAIL @ fw-upload (step 8) rc=-11`

  File changed:
    - `src/drivers/net/wifi_9260.c`

  Patch:
    - relaxed separator detection from `sec->size == 4` to `sec->size >= 4`
      while still requiring the marker dword to match a known separator value

  Effect:
    - 8-byte SEC_RT separator TLVs are now treated as boundaries
    - CPU1 stops at the CPU separator instead of consuming the full stream
    - CPU2 can load the next runtime image instead of failing with `-11`
