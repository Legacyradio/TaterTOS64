fry669 - 2026-03-09 00:10:00 CDT - WiFi 9260: align RFH doorbell and ALIVE interrupt phase

SESSION CONTEXT:
  User provided a Dell Precision 7530 WiFi handoff dump still failing at:
    - `wh`: step=9, rc=-1, `diag cpu-stuck-rx-no-used`
    - `wc`: `VERDICT: CPU STUCK`
    - RFH state: free-ring progress present (`RFH_R != 0`) but used-ring
      writeback still dead (`RFH_U = 0`, `rx_closed = 0`, `rx_finished = 0`)

WHY:
  After the earlier transport-shape fixes, the remaining mismatch was in the
  upload-to-ALIVE transition itself rather than in the ring layouts. Comparing
  the local bring-up to Linux 7.0-rc2 showed two concrete sequencing drifts:
    1. The driver was writing the live PRPH free-ring write index directly
       before ringing the RFH trigger doorbell.
    2. The direct FH upload path switched to a broad interrupt mask before the
       CPU-ready marker, instead of staying in the FH_TX-only phase until the
       ALIVE wait takes over.

  Both mismatches fit the observed signature where firmware starts, one IRQ is
  observed, the RFH free ring advances, but used-ring/status writeback never
  becomes visible to the host.

FILES CHANGED:
  - `src/drivers/net/wifi_9260.c`

CODE CHANGES:
  1. Stopped writing `RFH_Q0_FRBDCB_WIDX` directly during MQ RX bring-up.
     The driver now leaves the PRPH WIDX initialized at zero and advances the
     queue only through `RFH_Q_FRBDCB_WIDX_TRG(0)`, matching Linux's RFH
     restock doorbell behavior.

  2. Removed the early full interrupt-mask programming from
     `iwl_upload_runtime_cpu_image()`.
     The upload phase now remains in the FH_TX-only interrupt mode until
     `iwl_wait_alive()` explicitly rotates to the ALIVE/FH_RX handoff phase.

  3. Narrowed `iwl_wait_alive()` mask setup to `CSR_INT_BIT_ALIVE |
     CSR_INT_BIT_FH_RX` instead of pre-enabling the broader debug/error set.

RATIONALE:
  This is still a hardware-agnostic transport fix. It does not assume a Dell-
  specific quirk; it removes local sequencing drift from the Linux 9000-series
  upload/handoff path.
