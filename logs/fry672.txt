WHY:
  The bare-metal failure signature has been moving around between DMA setup,
  firmware upload, and ALIVE handoff. To stop random debugging:
    1. The driver now logs DMAR / DRHD / GCMD / GSTS state when attempting to
       disable Intel VT-d before allocating DMA memory.

       here as the canonical hardware triage flow.

FILES CHANGED:
  - src/drivers/net/wifi_9260.c

CODE CHANGES:
  1. Added Intel VT-d register definitions and minimal DMAR/DRHD parsing.
  2. Added `iwl_disable_intel_iommu()`:
     - locates ACPI `DMAR`
     - walks remapping entries
     - maps each DRHD MMIO register window uncached
     - clears `GCMD.TE`
     - waits for `GSTS.TES` to clear
     - logs per-DRHD state and a final disable summary
  3. Called the VT-d disable helper immediately before `iwl_dma_alloc()`.

RATIONALE:
  This is a hardware-agnostic direct-DMA bring-up step, not a Dell-only hack.
  It uses standard ACPI DMAR discovery and Intel VT-d global command/status
  bits, and leaves the driver running even if disable is incomplete so the
  next diagnostic layer can still be inspected.
