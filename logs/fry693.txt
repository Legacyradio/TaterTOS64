WHY THIS PATCH:
  This is the exact branch eventually4 said to take next.

  The ALIVE group/id fix in fry692 removed one protocol mismatch, but the
  hardware still died before usable RX/ALIVE handoff. That means the next
  highest-confidence bug is INIT/runtime sequencing, specifically that local
  code was still doing this:

    INIT boot -> INIT ALIVE -> wait INIT_COMPLETE
    runtime boot -> PHY_DB send -> PHY_CFG + NVM reads

  The bundled Linux Core43 MVM path instead does:

    INIT boot -> INIT ALIVE -> NVM work -> TX_ANT_CFG -> PHY_CFG
    -> wait INIT_COMPLETE / PHY_DB
    runtime boot -> TX_ANT_CFG -> PHY_DB send -> PHY_CFG

  So the local driver was still starting runtime without the INIT-side prep
  that Core43 expects to be complete already.

     New helper behavior:
       - derive valid TX/RX antenna masks from firmware TLV `phy_config`
       - send `TX_ANT_CONFIGURATION_CMD`
       - send `PHY_CFG_CMD` using TLV `phy_config` plus non-zero chain masks

  2. Moved NVM diagnostics and INIT calibration trigger into INIT phase
     Relevant lines after patch:
       - `3408-3487`
       - `5049-5063`

     New INIT-side order:
       1. boot INIT image
       2. wait INIT ALIVE
       3. read MAC / regulatory / PHY SKU NVM via `NVM_ACCESS_CMD`
       4. send `TX_ANT_CONFIGURATION_CMD`
       5. send `PHY_CFG_CMD`
       6. wait for `INIT_COMPLETE` / `PHY_DB`

     Important local safety point:
       - the NVM sync commands happen before `PHY_CFG_CMD`
       - that avoids `iwl_wait_resp()` consuming `PHY_DB` / `INIT_COMPLETE`
         notifications that `iwl_wait_init_complete()` needs to see later

  3. Kept runtime-side PHY DB replay in place and restored runtime PHY config
     Relevant lines after patch:
       - `5099-5109`
       - `3489-3512`

     Runtime-side order is now:
       1. boot runtime image
       2. send harvested PHY DB to runtime
       3. send `TX_ANT_CONFIGURATION_CMD`
       4. send `PHY_CFG_CMD`
       5. continue host-command/MAC init

     NVM reads were removed from the normal post-runtime path when INIT is
     present, except for a retry if INIT-phase MAC collection failed.
