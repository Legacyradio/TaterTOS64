WHY THIS PATCH:
  fry714 rewired the bring-up path to follow Linux 7.0-rc2 function
  boundaries more directly, but in doing so it dropped some earlier
  low-level cleanup writes that were still required on this platform.

  Local source comparison against Linux showed three concrete gaps:
    1. `SCD_QUEUE_STATUS_BITS()` was mapped incorrectly for queue ranges
       0..19 and 20..31.
    2. `iwl_pcie_tx_init()` no longer stopped all FH TX DMA channels before
       queue programming.
    3. The 9000-series RB timeout write (`HBUS_TARG_MBX_C = 0x64`) was no
       longer present on the active RX init path.
