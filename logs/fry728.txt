Phase one VM ABI: add guard pages, close phase one

WHAT:
- Added FRY_MAP_GUARD (0x100) flag for creating guard page regions
- Guard pages are tracked as FRY_VM_REGION_GUARD: no backing pages, no commit
  allowed, any userspace access faults with "guard" diagnostic and kills the
  process
- Added vm_map_guard_region() kernel path, hooked into SYS_MMAP
- Added fry_mguard() libc wrapper
- Extended vmtest with guard page coverage:
  - Allocate single guard page, verify mcommit is rejected, verify munmap works
  - Allocate 3-page guard region, verify munmap works
- Tests use exit codes 55-59

WHY:
- FRY_VM_REGION_GUARD was defined in process.h and handled in mprotect/fault
  paths but had no userspace-facing creation path — dead code in the ABI
- Guard pages are the standard mechanism for stack overflow detection and heap
  corruption fencing; the ABI is incomplete without them
- This is the last missing feature in phase one scope
