What changed:

  ACPI/AML startup after the first userspace handoff.

  which also meant the old inline code after `sched_tick()` no longer ran.
- A live environment still needs the deferred ACPI event worker and AML
  background startup, so that work now lives in a real kernel task instead of
  depending on an invalid resume path.
