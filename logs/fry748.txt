DESIGN DECISIONS:
  - Socket pool (g_sockets[16]) mirrors pipe pool pattern — global, indexed by fd_ptrs[]
  - TCP wraps existing netcore tcp_connect/send/recv; listen/accept add server-side
  - UDP uses fallback handler (udp_set_socket_handler) for packets not matching bindings
  - Nonblocking: O_NONBLOCK fd_flag + MSG_DONTWAIT flag both checked
  - Poll: FD_SOCKET returns POLLIN/POLLOUT/POLLHUP based on TCP state and buffer status
  - DNS: thin wrapper over netcore dns_resolve(), copies result to user buffer
