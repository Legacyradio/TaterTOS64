FILES CREATED:
  src/include/fry_time.h — shared time types (fry_timespec, clock IDs)
    Why: kernel + userspace need common time structures for clock_gettime/nanosleep

  src/include/fry_random.h — shared random constants (GRND_* flags, max len)
    Why: kernel + userspace need common getrandom flag definitions

  src/kernel/entropy/entropy.h — entropy subsystem header
  src/kernel/entropy/entropy.c — entropy subsystem implementation
    Why: crypto-grade random bytes for getrandom()
    Design:
      - CPUID detection of RDRAND (leaf 1, ECX.30) and RDSEED (leaf 7, EBX.18)
      - 256-bit internal pool mixed with SipRound-inspired CSPRNG
      - Primary: RDSEED → RDRAND fallback → RDTSC last resort
      - Re-seeds from RDRAND + RDTSC on every getbytes() call
      - Spinlock-protected for SMP safety

  src/drivers/timer/rtc.h — RTC driver header
  src/drivers/timer/rtc.c — CMOS RTC driver
    Why: wall-clock time for CLOCK_REALTIME
    Design:
      - MC146818-compatible CMOS read via ports 0x70/0x71
      - Double-read consistency check (no mid-update race)
      - BCD/binary auto-detect via Status Register B
      - 12/24-hour mode handling
      - Century register with fallback to 20xx
      - Rata Die epoch conversion (UTC)
      - Boot-time epoch captured once by rtc_init()

DESIGN DECISIONS:
  - Entropy pool uses SipRound-inspired mixing (rotate-xor-add) for speed
    and cryptographic strength with RDRAND input
  - CLOCK_REALTIME = rtc_boot_epoch + HPET monotonic offset (no NTP/drift
    correction — appropriate for a standalone OS)
  - CLOCK_BOOTTIME is alias for CLOCK_MONOTONIC (no suspend state)
  - nanosleep rounds sub-ms sleeps up to 1ms (scheduler tick granularity)
  - FRY_RANDOM_MAX = 256 bytes per call (prevents kernel stack abuse)
