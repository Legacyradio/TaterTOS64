DESIGN DECISIONS:
  - FAT32 is the writable profile filesystem for first browser port
    Reason: FAT32 already had read/write support; extending it is more
    productive than enabling ToTFS or adding write support to NTFS
  - 8.3 short names for create/rename (LFN creation not implemented)
    Reason: LFN entry generation is complex; 8.3 is sufficient for
    programmatic file management (profiles, caches, temp files)
  - Truncate only supports shrinking (not extending beyond current size)
    Reason: write() already extends files; truncate-down covers the common
    case of file rewrite/recycle
  - Rename supports cross-directory moves (updates ".." for directories)
  - vfs_rename enforces same-mount constraint (no cross-FS rename)
  - SYS_FSTAT extracts attr from embedded fat32_file in vfs_file.private[]
    This is safe because sizeof(fat32_file) < sizeof(vfs_file.private)
