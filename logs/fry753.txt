DESIGN DECISIONS:
  - Rich key events are delivered via SYS_KBD_EVENT syscall (not fry_read(0))
    Reason: fry_read(0) remains ASCII-only for backward compatibility; new
    rich path provides scancode, VK, modifiers, and press/release state
  - Intellimouse wheel detection via magic sequence (200/100/80 sample rates)
    Reason: standard PS/2 extension protocol; QEMU, VMware, and real hardware
    all respond correctly to this sequence
  - Clipboard stored in kernel (4096 byte buffer)
    Reason: simplest architecture — apps access via syscall, compositor
    proxies via TaterWin messages; kernel buffer persists across app restarts
  - Cursor shapes are 8x16 1-bit bitmaps (6 shapes)
    Reason: simple, no alpha compositing needed; sufficient for browser
    (arrow, ibeam for text, hand for links, resize, crosshair, wait)
  - Damage tracking is coarse (union of dirty rects per window per frame)
    Reason: full dirty-rect compositing is complex; this is the first step.
    VRAM copy is already conditional on changes.
  - Close request is sent before kill on close button click
    Reason: apps that handle TW_MSG_CLOSE_REQUEST can save state before exit
