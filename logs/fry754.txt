DESIGN DECISIONS:
  - FILE streams use 4KB internal buffers, not user-supplied buffers
    Reason: simpler implementation; setvbuf accepts but ignores user buffer
    parameter; sufficient for browser config/profile I/O

  - errno uses TLS key for per-thread storage
    Reason: NSPR and NSS depend on thread-safe errno; falls back to global
    if TLS not initialized (main thread before first TLS init)

  - Signal handlers are stored but never delivered
    Reason: TaterTOS has no signal mechanism yet; storing handlers lets
    NSPR's signal(SIGPIPE, SIG_IGN) compile and not crash

  - pthread_create uses a heap-allocated context + trampoline
    Reason: pthreads return void*, fry_thread_create takes void(*)(void*);
    trampoline adapts calling convention cleanly without function pointer casts

  - Math library uses software algorithms (Taylor, Newton-Raphson)
    Reason: no assumption of x87 FPU or SSE beyond basic arithmetic;
    sufficient accuracy for browser/crypto use cases

  - dlopen returns sentinel (1) for NULL filename, NULL for all others
    Reason: NSPR uses dlopen(NULL) to get handle to self for dlsym;
    returning non-NULL sentinel lets it call dlsym which safely returns NULL

  - Time functions are UTC-only
    Reason: TaterTOS has no timezone database; localtime == gmtime;
    sufficient for cert validation and NSPR time APIs

  - getaddrinfo returns both SOCK_STREAM and SOCK_DGRAM results
    Reason: standard behavior when ai_socktype is 0; NSPR iterates results
