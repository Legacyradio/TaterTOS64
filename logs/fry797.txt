fry797 — BearSSL swap complete — mbedTLS replaced
=====================================================================
Timestamp: 2026-04-02

CHANGE:
  Replaced mbedTLS with BearSSL as the TLS library for TaterSurf.
  Complete rewrite of the TLS integration layer in ts_http.h.

  BearSSL advantages over mbedTLS for TaterTOS bare-metal:
    - Zero dynamic allocation (all buffers caller-provided)
    - Zero global state (all state in caller-owned structs)
    - Designed for embedded/bare-metal
    - TLS 1.2 only (sufficient — Google/most sites still accept)

WHY:
  mbedTLS crashed on the 2nd TLS read operation on any connection,
  regardless of TLS version (1.2/1.3) or crypto backend (PSA/legacy).
  RIP=0x0 deep in ssl_fetch_input. After 10+ trace iterations
  (fry793-795), every mbedTLS-internal theory was ruled out. The crash
  was a fundamental incompatibility between mbedTLS 3.x and TaterTOS's
  bare-metal environment.

INTEGRATION APPROACH:
  - BearSSL low-level engine API (non-blocking, poll-based)
  - Custom X.509 "no-validate" wrapper: delegates to br_x509_minimal
    for server public key extraction, but overrides end_chain() to
    always return 0 (accept all certificates, same as mbedTLS VERIFY_NONE)
  - Entropy from fry_getrandom() injected via br_ssl_engine_inject_entropy()
  - BR_SSL_BUFSIZE_BIDI (~33KB) static buffer per connection
  - Engine pump functions (ts_tls_pump_send/recv) called each poll cycle
