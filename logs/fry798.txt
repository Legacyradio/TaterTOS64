=====================================================================

FIXES (on top of fry797 BearSSL swap):

  1. X.509 time callback — return 0 not 1
     Problem: br_x509_time_check callback must return 0 for "within
     validity range". Returning 1 means "after notAfter" (EXPIRED).
     Without a working time callback, x509_minimal gets TIME_UNKNOWN (53)
     or EXPIRED (54) and aborts key extraction early — pkey.key_type=0.
     Fix: ts_xnv_time_check() always returns 0.

  2. get_pkey error bypass
     Problem: xm_get_pkey() only returns the key if err is OK (32) or
     NOT_TRUSTED (62). With 0 trust anchors, end_chain sets err to
     NOT_TRUSTED, but other errors (TIME_UNKNOWN, EXPIRED) can also
     occur and prevent key return.
     Fix: ts_xnv_get_pkey() forces inner.err = BR_ERR_X509_NOT_TRUSTED
     before calling inner get_pkey, ensuring the key is always returned.
