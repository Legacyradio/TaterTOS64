STEP 17: HTTP improvements (ts_http.h + ts_dom_bindings.h)
-----------------------------------------------------------

17a) User-Agent overhaul:
  - OLD: "TaterSurf/1.0 (TaterTOS64v3)"
  - NEW: "Mozilla/5.0 (TaterTOS64v3) AppleWebKit/537.36 (KHTML, like Gecko)
          TaterSurf/2.0 Chrome/120.0.0.0"
  - Matches Chrome UA format — prevents sites from serving degraded content
    or blocking "unknown browser" UAs
  - Updated in both ts_http.h (request headers) and ts_dom_bindings.h
    (navigator.userAgent)

17b) Request headers improved:
  - Accept: added application/xml;q=0.9, application/json
  - Accept-Language: en-US,en;q=0.9 (was missing — some sites require it)
  - Upgrade-Insecure-Requests: 1 (hints HTTPS preference)
  - Sec-Fetch-Dest: document (prevents "suspicious request" blocks)
  - Sec-Fetch-Mode: navigate
  - Sec-Fetch-Site: none
  - Connection: keep-alive (was already present)

17c) HTTP/2 status:
  Full HTTP/2 implementation requires:
  - ALPN negotiation in TLS handshake (BearSSL supports it)
  - Binary frame parser (DATA, HEADERS, SETTINGS, etc.)
  - HPACK header compression (Huffman + dynamic table)
  - Multiplexed streams on single connection
  - Flow control per stream
  This is a ~2000-3000 line addition. Current HTTP/1.1 works with
  all sites that accept it. Most sites serve HTTP/1.1 to clients
  that don't negotiate h2 via ALPN. Future session work.
