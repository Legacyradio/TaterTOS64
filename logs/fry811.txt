fry811 — TaterSurf blank page fix + cursor outline + GIF decoder
Date: 2026-04-03

== Summary ==
Fixed white/blank page rendering in TaterSurf after page load.
Added black outline to GUI mouse cursor so it's visible on all backgrounds.
Added GIF image decoder to ts_image.h.
Removed destructive IMGREBUILD (full HTML re-parse after image load).

== Root Cause: White Page ==
After DOM_SYNC, ts_doc_init() resets page_bg to TS_COL_BG (0x0E131B).
Then ts_doc_build_from_dom() hits <body>, sees page_bg == TS_COL_BG,
and forces page_bg = 0xFFFFFF (white). Text color is 0xE6ECF8 (near-white)
→ invisible on white background.

Fix: Removed the forced-white body default in BOTH code paths:
  - ts_layout.h:1462-1465 (tokenizer path)
  - ts_layout.h:2422-2423 (DOM rebuild path)
Dark theme now persists when no CSS background is specified.

== Root Cause: Page Goes Blank After Image Load ==
After image download completes, tatersurf.c had an "IMGREBUILD" block
(lines ~992-1017) that did a FULL HTML re-parse via ts_doc_build().
This re-parse would trigger another DOM_SYNC that replaced 81-node
tokenizer tree with 41-node DOM tree (mostly BREAKs). The page content
would vanish.

Fix: Replaced full re-parse with just ts_doc_layout() to reposition
existing nodes with newly decoded images. No re-parse needed.

== Cursor Fix ==
gui.c draw_cursor(): was solid white (0xFFFFFF) only — invisible on
white backgrounds. Added black (0x000000) outline drawn at +1px offset
before the white fill.

== GIF Decoder ==
Added ts_gif_decode() and ts_gif_lzw_decode() to ts_image.h.
Supports GIF87a/89a, single frame, global/local color tables,
interlacing, transparency. LZW decompression with 4096-entry table.
Added GIF detection to ts_image_decode() (checks "GIF" magic bytes).

== Font Status ==
FONT: init=0 loaded=0 — DejaVu TTF at /fonts/DEJAVU.TTF not found
by fopen() at runtime despite being in ISO staging. Bare-metal
filesystem may not mount /fonts/ path. Bitmap 8x16 fallback is used.
NOT a blocker — bitmap text renders correctly.
