OUTSTANDING UNKNOWNS (will resolve at first compile failure)
==========================================================================
1. AK includes from libstdc++:
     <initializer_list> <iterator> <memory> <new> <utility>
   Verified in fry816 §1. The cross-compiler libstdc++ headers will
   either work (D3 option A) or pull in glibc paths and need a
   vendored 5-header replacement (D3 option B). Determined in fry824.

2. AK __cxa_demangle:
   Demangle.h calls libstdc++'s __cxa_demangle. If the freestanding
   cross-compiler libstdc++ does not provide it, we either link
   libsupc++ (LGPL with runtime exception, header-only impl
   inadvisable) OR write a TaterTOS DemangleTaterTOS.cpp that
   returns the mangled name unchanged (acceptable fallback — only
   used in panic/backtrace pretty-printing). Decision at first
   compile failure.

3. Backtrace.h.in:
   AK/Backtrace.h.in is a CMake-configured stub that exposes
   AK::Backtrace. TaterTOS has no unwinder yet (D2 = -fno-exceptions
   means no __builtin_return_address chain machinery via libgcc_eh),
   but x86_64 frame-pointer walking through known-RBP saves is
   trivially implementable. We implement Backtrace.h directly
   when AK/Format.cpp first #includes it. Logged separately.

4. AK kmalloc.cpp:
   AK has its own kmalloc / kfree wrappers for use INSIDE Ladybird
   processes (not the kernel-side TaterTOS kmalloc). The wrappers
   call malloc / free, which TaterTOS libc.c provides. Should work
   first try.

5. LibCore/Process.cpp uses posix_spawn / spawn_h:
   TaterTOS libc.h doesn't expose posix_spawn directly. Existing
   fry_spawn / fry_spawn_args are the underlying primitives. May
   need a thin wrapper in libc.c or in Process.cpp. Decision at
   first compile failure.

6. LibCore/EventLoopImplementationUnix.cpp uses pselect/ppoll/
   eventfd/timerfd_create:
   TaterTOS exposes fry_poll. eventfd / timerfd are Linux-specific.
   This is the biggest unknown — the Unix event loop may need a
   TaterTOS-tailored EventLoopImplementationTaterTOS.cpp companion
   to replace eventfd usage with futex-based wakeup. We discover
   exactly which Linux-isms are touched at fry824 compile time.
