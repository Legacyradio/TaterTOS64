LibCore — LINKED LIBRARIES (line 94)
==========================================================================
  target_link_libraries(LibCore PRIVATE
      LibUnicode
      LibURL
      LibTextCodec
      Threads::Threads)

LibCore has THREE Ladybird-internal sister libraries as link deps:

  LibUnicode    — 20 .cpp files (~ICU wrapper + locale)
  LibURL        — 5 .cpp files
  LibTextCodec  — 2 .cpp files

Plus pthreads (Threads::Threads), already provided by TaterTOS libc.

This means Chunk 2 ("AK + LibCore build clean") cannot stop at
LibCore — it must also bring in LibUnicode + LibURL + LibTextCodec
to resolve symbols at link time. Otherwise LibCore.a builds (it's a
static lib, no link enforced), but anything that LINKS against
LibCore.a breaks immediately.

This is fry821 §"OUTSTANDING UNKNOWNS" #1 graduating from "noted"
to "definitively required."

THE TWO SURPRISES (vs fry821)
==========================================================================

S1. mimalloc dependency.
    AK's kmalloc.cpp routes allocations through mimalloc instead of
    libc malloc. mimalloc is ~30 .c files (~10k LOC), MIT licensed,
    portable C with platform abstractions for Linux/macOS/Windows/BSD.
    No TaterTOS port exists.

    OPTIONS (decision needed):
      (S1a) Vendor mimalloc, port its OS abstractions to TaterTOS.
            Real engineering work. Adds a new high-quality allocator
            to TaterTOS. Estimated ~1-2 days for the OS-shim layer
            (mimalloc has clean abstraction boundaries — see its
            src/prim/ subdirectory).
      (S1b) Engineer around it: patch AK's kmalloc.cpp to call
            TaterTOS libc malloc/free directly. This means
            maintaining a small TaterTOS-specific divergence from
            upstream AK. Existing TaterTOS libc.c has malloc/free/
            calloc/realloc/aligned_alloc/posix_memalign/memalign/
            valloc/pvalloc (verified in fry819).

    Recommendation: S1b. Reason consistent with the TaterLabs ethos
    (build the road, don't follow the map): TaterTOS already has a
    working allocator that other apps use. Adding a second allocator
    on top is mapping-following. The divergence is one #ifdef in
    AK/kmalloc.cpp — clearly marked, easy to upstream as a config
    option later if we ever want to.

S2. LibCore's transitive deps are mandatory at link time.
    LibUnicode + LibURL + LibTextCodec must be ported in Chunk 2,
    not deferred. Total = 27 additional .cpp files.

    LibUnicode in particular wraps ICU. Two paths:
      (S2a) Vendor ICU. ICU is huge (~30MB of source).
      (S2b) Use a slimmed-down LibUnicode that backs onto a smaller
            Unicode lib (utf-8, character classes, normalization,
            case mapping). Several upstream Ladybird configs do this
            for embedded targets.
      (S2c) Provide TaterTOS-native LibUnicode shims that return
            ASCII-correct results for everything Unicode and accept
            that BiDi / complex shaping won't work in v1. Real
            shaping is a v2 concern.

    Recommendation: defer to fry823 (LibUnicode investigation) for
    the actual call. This needs reading LibUnicode's CMakeLists +
    file list to see how much of it is ICU-glue vs internal logic.
