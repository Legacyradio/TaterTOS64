LIBRARY SHAPE — internal amalgamation
=====================================================================
simdutf differs in shape from fmt (header-only + one .cc) and
fast_float (header-only). It uses its OWN amalgamation pattern:

  src/simdutf.cpp (156 LOC) is a single TU that #includes every
  kernel-specific .cpp/.h:
    arm64/      icelake/    haswell/    westmere/   ppc64/
    rvv/        lsx/        lasx/       fallback/   scalar/
    generic/    tables/

  Preprocessor + macro guards select only the relevant kernels for
  the target. On TaterTOS x86_64 we'll compile westmere + haswell +
  icelake + fallback + scalar + generic; arm64/ppc64/rvv/etc. paths
  are macro-disabled.

Build integration is therefore:
  - Compile ONE translation unit: src/simdutf.cpp → simdutf.o
  - TWO include paths needed:
      -I src/user/apps/ladybird-3rd/simdutf/include   (public API)
      -I src/user/apps/ladybird-3rd/simdutf/src       (internal)
