DIFF 1: New file Backtrace.h (rendered from Backtrace.h.in)
-----------------------------------------------------------
TaterTOS does not link libbacktrace and does not vendor cpptrace.
The render leaves AK_HAS_BACKTRACE_HEADER undefined, so AK::Backtrace
falls back to its no-libbacktrace path. File contents = a comment
block explaining provenance. No active code.

DIFF 2: New file Debug.h (rendered from Debug.h.in)
----------------------------------------------------
Generated by the sed transformation:
  s/^#([[:space:]]*)cmakedefine01 (.*)$/#\1define \2 0/
  s/^#([[:space:]]*)cmakedefine (.*)$/\/* #\1undef \2 (off on TaterTOS) *\//

Verification: zero remaining 'cmakedefine' substrings in the output
(grep -c 'cmakedefine' Debug.h == 0).

Result: every Ladybird debug channel (~88 of them: AUDIO_DEBUG,
BMP_DEBUG, CACHE_DEBUG, CSS_LOADER_DEBUG, etc.) is set to 0. To
enable a single channel locally, override at compile time with
-DXXX_DEBUG=1.

DIFF 3: kmalloc.cpp patch (S1b — replace mimalloc with libc malloc)
--------------------------------------------------------------------
The minimal-touch pro patch: rename
AK_USE_SYSTEM_ALLOCATOR_INSTRUMENTED -> AK_USE_SYSTEM_ALLOCATOR (so
the macro name is honest for both the original sanitizer use and our
TaterTOS use), and add `defined(AK_OS_TATERTOS)` to its trigger.

Lines changed:
  Line 76 (BEFORE):
    #    if __has_feature(address_sanitizer) || defined(__SANITIZE_ADDRESS__)
  Line 76 (AFTER):
    #    if defined(AK_OS_TATERTOS) || __has_feature(address_sanitizer) || defined(__SANITIZE_ADDRESS__)

  Line 79 (BEFORE):
    #        define AK_USE_SYSTEM_ALLOCATOR_INSTRUMENTED 1
  Line 79 (AFTER):
    #        define AK_USE_SYSTEM_ALLOCATOR 1

  Line 89 (BEFORE):
    #    ifdef AK_USE_SYSTEM_ALLOCATOR_INSTRUMENTED
  Line 89 (AFTER):
    #    ifdef AK_USE_SYSTEM_ALLOCATOR

Plus a TaterTOS-specific comment block above the conditional pointing
to logs/fry823.txt and logs/fry827.txt.

NET EFFECT: when the TaterTOS Makefile defines `-DAK_OS_TATERTOS=1`
(fry828 will add this), AK::kmalloc routes calloc/malloc/realloc/
free/posix_memalign through the libc symbols already implemented in
src/user/libc/libc.c. No mimalloc dependency. Same code path also
satisfies sanitizer builds (upstream's original case), unchanged.

The patch is upstreamable as a tiny PR if Ladybird ever wants to
clean up the macro name. Zero behavioral change for non-TaterTOS,
non-sanitizer builds.

WHY THE BUILD-CONTEXT DIAGNOSTICS ARE EXPECTED
=====================================================================
Clang LSP, analyzing kmalloc.cpp in isolation (without our build
flags), reports two warnings:

  Line 8:  'AK/kmalloc.h' file not found
    Reason: LSP has no `-I src/user/apps/ladybird` on its include
    path. fry828 will add that flag to the Makefile. Compile-time
    invocation will resolve <AK/kmalloc.h> correctly because the
    file exists at src/user/apps/ladybird/AK/kmalloc.h (we copied it
    above).

  Line 87: 'mimalloc.h' file not found
    Reason: that #include is in the `#else` branch — only reached
    when AK_OS_TATERTOS is undefined AND no sanitizer is active. On
    TaterTOS builds the Makefile sets -DAK_OS_TATERTOS=1, so the
    line is preprocessed-out and never seen by the compiler. The
    file does not need to exist on disk. Identical situation in
    upstream Ladybird outside vcpkg context.

Neither will affect the actual build. Documented here so a future
engineer doesn't chase them as bugs.
