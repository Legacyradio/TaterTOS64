What AK uses unistd.h for here: just sysconf(_SC_PAGESIZE) to set

WHY THIS HAPPENED
=====================================================================
TaterTOS's userland libc has the POSIX surface (posix.c, libc.c)
but does NOT expose a top-level `<unistd.h>` header in src/include/.
The functions are reachable via #include <libc.h> instead.

This was pre-flagged in fry821 §"OUTSTANDING UNKNOWNS" #5:
  "LibCore/Process.cpp uses posix_spawn / spawn_h: TaterTOS libc.h
   doesn't expose [these] directly. May need a thin wrapper..."
unistd.h falls in the same class.

FIX OPTIONS (need Zack call before fry829 implements one)
=====================================================================
F1. Engineer a real src/include/unistd.h that forwards to libc.h.
    This is the road-building move per the TaterLabs ethos. Every
    TaterTOS app that ports POSIX-flavored code (Ladybird now,
    others later) can rely on the standard <unistd.h> path.
    Content: minimal POSIX-correct unistd.h declarations forwarded
    to TaterTOS's existing fry_* / posix_* primitives. Real code,
    not a stub. ~50-100 lines to start; grows as later AK/LibCore
    files demand more symbols (each addition logged).

    Sister headers we'll likely need on the same path:
      <fcntl.h>     (open flags — partly in fry_fcntl.h already)
      <sys/stat.h>  (struct stat — fry_stat exists, no header)
      <sys/mman.h>  (mmap — fry_mmap exists, no header)
      <dirent.h>    (closedir/opendir/readdir — posix.c has them)
      <pthread.h>   (pthread.c implements; no header — yet)
      <sys/socket.h><netinet/in.h><arpa/inet.h>
                    (fry_socket.h has TaterTOS-native types; need
                     POSIX-named aliases for ports)

    These would be new files under src/include/ created one-at-a-time
    in their own fry logs as AK/LibCore demands each.

F2. Patch AK/Platform.h's PAGE_SIZE block to use a TaterTOS-native
    constant.  TaterTOS pagesize is 4096 (verified in
    src/include/fry_limits.h or similar). Replace lines 274-283
    with a TaterTOS-aware branch:
       #elif defined(AK_OS_TATERTOS)
       #    define PAGE_SIZE 4096
    Smaller patch, but kicks the can down the road — AK/LibCore have
    many other unistd.h consumers. Each will hit the same wall.

    NOT recommended. We'd just be writing a one-off patch per call
    site. F1 fixes the entire class.

F3. Hybrid: F1 + F2 together. Engineer src/include/unistd.h with
    just the symbols AK actually uses on this code path (sysconf,
    _SC_PAGESIZE), AND add an AK_OS_TATERTOS branch in Platform.h
    that bypasses the runtime sysconf call with a compile-time 4096
    (since TaterTOS pagesize is fixed and known at compile time —
    micro-optimization avoiding a sysconf call).

    This is the pro version. Real header, real implementation, and
    a well-marked targeted bypass at the AK level for the one place
    where compile-time knowledge beats runtime knowledge.

    Recommendation: F3.
