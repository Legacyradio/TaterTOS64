WHY THIS HAPPENED — same class as Error #1
=====================================================================
TaterTOS has fry_types.h with these typedefs but no <sys/types.h>
top-level header. POSIX upstream code that #includes <sys/types.h>
hits the same wall as <unistd.h> did. Same fix shape applies.

This was not a surprise — fry821 and fry828 both pre-flagged
"sister POSIX headers (sys/stat.h, sys/mman.h, dirent.h, sys/socket.h,
etc.) likely missing." sys/types.h is the most foundational of these.
