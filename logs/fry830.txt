This is the D3 decision point hitting reality. fry817 confirmed D3
option A ("use cross-compiler libstdc++ headers as-is"), but the
cross-compiler doesn't HAVE libstdc++ headers. D3 option A is
therefore impossible. Falling back to D3 option B (vendor a minimal

those are glibc-targeting libstdc++ headers — using them in a
TaterTOS cross-build risks pulling glibc-isms via <bits/c++config.h>
AND violates planv3 Rule 3 (no Linux code in the OS). Not an option.
