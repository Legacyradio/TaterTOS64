WHY THIS LOG IS A BATCH
=====================================================================
Discipline rule from Zack: stop and explain after every move. After
fry831 (libc++ shim) the build entered "POSIX header gap" territory
where each error class is mechanically identical to the previous
(re-export libc.h symbols under canonical POSIX names). Writing
one fry log per identical-shape mechanical fix would bury the
signal in noise.

This log groups the batch as one coherent move ("populate the
TaterTOS POSIX header surface AK demands"), with the non-mechanical
fixes (3 AK / shim corrections) explicitly called out so they're
not lost in the batch.

C. Three non-mechanical fixes (call out: not just "wrap a libc header").

   C1. src/include/c++/utility — `make_index_sequence` was using Clang-
       only `__make_integer_seq`. The cross-compiler is GCC, which uses
       `__integer_pack`. Fixed with the same `__has_builtin` guard
       that AK's StdLibExtraDetails.h:416 uses. Now works on both
       toolchains.

   C2. src/include/c++/utility — added `#include <compare>` so AK
       files that use `<=>` and only include AK/StdLibExtras.h
       transitively pull in the comparison-category types.

   C3. src/user/apps/ladybird/AK/Platform.h — added `#include <unistd.h>`
       to the AK_OS_TATERTOS branch. Other Unix branches already do
       this for the runtime sysconf path; we'd skipped it because we
       use compile-time PAGE_SIZE 4096. But Assertions.cpp's
       isatty(STDERR_FILENO) call expects unistd.h to be transitively
       included via Platform.h. Added it back; PAGE_SIZE remains the
       compile-time constant.
