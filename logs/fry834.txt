C. PATCHED files (engineering decisions, not just header re-exports)

   C1. src/user/libc/libc.h
       Wrapped struct tm + typedef int64_t time_t + #define
       CLOCKS_PER_SEC in include guards (_TATERTOS_STRUCT_TM_DEFINED,
       _TATERTOS_TIME_T_DEFINED). Lets src/include/time.h define the
       same shapes without conflict — whichever header is included
       first wins.

   C2. src/include/time.h
       Same struct tm guard. Identical typedef redeclarations are
       legal C/C++ provided the typedef target matches; we ensure
       both headers declare exactly `typedef int64_t time_t`.

   C3. src/include/c++/string
       Added `#include <string_view>` so simdutf's error.h (which
       references std::string_view without #include <string_view>)
       compiles. libstdc++ and libc++ both pull string_view from
       string transitively; we match the pattern.

   C4. src/include/errno.h
       Added 22 POSIX errno values commonly referenced by
       AK/LibCore/upstream code: EOVERFLOW, ELOOP, ENOMSG, EIDRM,
       ENODATA, EBADMSG, ENOTUNIQ, EILSEQ, ENOTSUP, EOPNOTSUPP (=ENOTSUP),
       EHOSTDOWN, ESTALE, EDQUOT, ECANCELED, EOWNERDEAD,
       ENOTRECOVERABLE, EXDEV, EROFS, EMLINK, EDOM, ENOEXEC, EFBIG,
       EDEADLK, EDEADLOCK (=EDEADLK), ENOLCK. Values match Linux.

   C5. src/user/apps/ladybird/AK/Random.cpp
       AK's csprng() had platform branches for Serenity/Android/BSD/
       Haiku/glibc-2.36+/Linux/Windows and a static_assert(false) on
       all others — fired on TaterTOS. Added an AK_OS_TATERTOS branch
       that calls fry_getrandom() (canonical TaterTOS entropy source,
       backed by the kernel CSPRNG pool in src/kernel/entropy/).
       Marked as "TaterTOS divergence" with logs/fry834.txt pointer.
       The fry_getrandom contract differs from Linux getrandom in a
       useful way: no EINTR loop required (TaterTOS doesn't deliver
       async signals to userland during this syscall), so the
       TaterTOS branch is simpler than the Linux one.

D. (carried-over fix from earlier in pass)
   src/include/c++/utility — added `#include <compare>` so AK's
   defaulted operator<=> deduces std::strong_ordering correctly.
   (Recorded in fry832, restated here for completeness.)
