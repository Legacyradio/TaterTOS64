C. PATCHES
   C1. src/include/assert.h  — Wrap the `#define static_assert _Static_assert`
       in `#if !defined(__cplusplus)`. In C++ static_assert is a keyword and
       must NOT be macro-defined. fast_float tripped on this — its
       static_assert(is_supported_float_type<T>::value, ...) was being
       rewritten to `_Static_assert` which doesn't exist as an identifier.

   C2. Makefile AK_CXXFLAGS  — Added:
         -U__STDCPP_FLOAT64_T__  -U__STDCPP_FLOAT32_T__
         -U__STDCPP_FLOAT16_T__  -U__STDCPP_BFLOAT16_T__
       gcc 15.2 freestanding defines all four C++23 fixed-width float
       feature macros. fast_float gates branches on them and references
       std::float64_t / float32_t / float16_t / bfloat16_t — types we
       don't (and won't, for v1) provide. Undefining the macros at
       compile time cleanly bypasses those branches without modifying
       fast_float source. TaterTOS does not currently expose IEEE-754-
       guaranteed fixed-width float types to userland.

CURRENT BLOCKERS (fry836 will fix)
=====================================================================
fmt/format.h needs:
  1. std::basic_string<CharT, Traits, Allocator> template
     (we have only std::string as a non-template class).
  2. std::system_error (exception subclass with errc/error_code).
  3. std::decay<T> type trait.

These are real engineering work, not mechanical shims:
  - basic_string requires turning our std::string class into a
    template specialization. Or providing basic_string as a separate
    template that happens to alias to string for char.
  - system_error requires std::error_code, std::error_category, plus
    the system_error class itself. Manageable since we use
    -fno-exceptions; nothing actually throws.
  - decay is mechanical (combines remove_reference + remove_cv +
    array-to-pointer + function-to-pointer).
