A. STRATEGY PIVOT — fmt switched to FMT_HEADER_ONLY
   Reason: fmt's format.cc translation unit demanded an ever-deepening
   pile of std:: stdlib surface (iterator_traits, basic_string::assign,
   allocator_traits, std::move, std::fprintf, std::char_traits<char>::
   to_int_type, ambiguous FMT_COMPILE_STRING vs fstring, etc.). Each
   fix unblocked one site and revealed two more.

   FMT_HEADER_ONLY is upstream-supported. fmt's templates inline into
   each consumer TU; format.cc is not compiled. AK already includes
   fmt's headers in its own .cpp files (Format.cpp, NumberFormat.cpp,
   etc.) — those compiled cleanly in fry836. So shipping fmt
   header-only is exactly equivalent to AK's header-only fmt usage,
   just with the heavy machinery duplicated across AK TUs (linker
   weak-symbol dedup handles the duplicate definitions).

   Changes:
     Makefile AK_CXXFLAGS += -DFMT_HEADER_ONLY=1
     AK_OBJS no longer includes $(LADYBIRD_3RD)/fmt/src/format.o
     The fmt format.cc compile rule is dead code (Make won't build it
       since it's no longer in any target's prereq list).

B. POSIX shim header additions for fmt + simdutf
   src/include/c++/array          ~50 LOC. std::array<T, N> for
                                  simdutf base64_tables.
   src/include/c++/iostream       Empty stub. simdutf's
                                  scalar/base64.h #include's it but
                                  uses no symbols.
   src/include/c++/tuple          ~100 LOC. std::tuple template +
                                  make_tuple + get<N>(t) +
                                  tuple_element + tuple_size. simdutf
                                  uses 3-element tuples for utf16/32
                                  conversion result types.

C. PATCHES — string template surface expansion
   src/include/c++/string         basic_string<CharT, Traits, Allocator>
                                  gained: assign(count, ch), assign(s, n),
                                  assign(s), append(s, n), append(s),
                                  append(other), append(count, ch),
                                  operator+=(ch), operator+=(s),
                                  operator+=(other), push_back(ch).
                                  Plus char_traits<CharT> gained:
                                  to_int_type, to_char_type, eof,
                                  eq_int_type, not_eof, compare,
                                  length, find, move, copy, assign-buf.

D. PATCHES — type_traits / utility cross-include hygiene
   src/include/c++/type_traits    Pull <iterator>, <cstdio>, then
                                  later removed (string handles
                                  these). Guarded remove_reference
                                  with _UTILITY_REMOVE_REFERENCE_DEFINED
                                  so utility and type_traits don't
                                  conflict.
   src/include/c++/utility        Same guard. Renamed local
                                  is_lvalue_reference to
                                  _utility_is_lvalue_ref to avoid
                                  collision with type_traits's
                                  is_lvalue_reference_<T&> partial.
                                  Added _utility_detail::decay_local
                                  so make_pair can use decay-style
                                  type stripping without including
                                  <type_traits> (would be circular).
   src/include/c++/string         Added transitive #include <memory>
                                  and <utility> to provide
                                  allocator_traits and std::move
                                  visibility (libstdc++ does this).

E. PATCH — pair conversion-copy ctor
   src/include/c++/utility        std::pair gained two new ctors:
                                    pair(pair<U1, U2> const&)
                                    pair(pair<U1, U2>&&)
                                  with element-wise conversion.
                                  simdutf's
                                    return std::make_pair(nullptr, ptr);
                                  needs pair<nullptr_t, X*> →
                                  pair<char32_t const*, X*> conversion.

F. PATCH — make_pair uses decay
   src/include/c++/utility        Replaced make_pair's
                                  remove_reference_t<T> with the
                                  inline decay_local. Matches libstdc++.

G. PATCH — Makefile AR path
   AR was set to $(TOOLBIN)/x86_64-elf-ar but the AUR
   x86_64-elf-binutils package installs ar at /usr/bin/x86_64-elf-ar.
   Fixed.

H. ASSORTED type_traits additions earlier in the build
   is_assignable, is_copy_assignable, is_move_assignable,
   is_trivially_copy/move_constructible, is_nothrow_*,
   is_member_pointer / is_member_function_pointer / is_member_object_pointer,
   is_base_of, is_convertible (SFINAE-based, gcc-portable).
