Build now stops on EventLoopImplementationUnix.cpp because the AK
HashMap<pthread_t, ThreadData*> needs an AK::Traits<pthread_t>
specialisation that doesn't exist. pthread_t in TaterTOS is a
struct (containing fry_thread + ctx), not a primitive integer like
glibc's pthread_t. AK's default Traits<T> requires hash + equality,
which the struct doesn't provide.
