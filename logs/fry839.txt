     struct Traits<::pthread_t> : public DefaultTraits<::pthread_t> {

D. libc.h EXTERN "C" WRAP (real engineering)
   The shim headers (stdlib.h, string.h, stdio.h, etc.) wrap their
   declarations in `extern "C"` for C++ consumers. libc.h had no
   such wrap, causing linkage mismatch. Surgical fix: wrap libc.h's
   C function declarations in `extern "C"`, and put the C++ helper
   overloads (operator==/!= for pthread_t) in an interleaved
   `extern "C++"` block.

   Layout:
     #ifdef __cplusplus
     extern "C" { #endif
       <C function decls>
     #ifdef __cplusplus
     } extern "C++" {
       <C++ operator overloads>
     } extern "C" { #endif
       <more C decls>
     #ifdef __cplusplus
     } #endif

E. PATCHES — operator==/!= overloads in libc.h
   Added pthread_t == int / int == pthread_t / != variants, since
   upstream code does `s_thread_id == 0` (Linux glibc has pthread_t
   as unsigned long). Maps integer-zero to "tid == 0" which is the
   uninitialised state.
