DIFFS APPLIED IN fry840
=====================================================================

A. unistd.h: full POSIX-fs surface batch (~70 extern decls)
   - Filesystem ops upstream LibCore expects:
     link, linkat, unlinkat, mknod, mkfifo, pipe2, pause, nice,
     alarm, acct, sethostname, getdomainname, setdomainname, syscall
   - Process group / session control:
     getsid, setsid, getpgrp, setpgid, getpgid, tcgetpgrp,
     tcsetpgrp, ttyname, ttyname_r, getlogin, getlogin_r
   - Temp file creation:
     mkstemp, mkstemps, mkdtemp, mktemp, tmpnam, mkostemp, mkostemps
   - POSIX-2008 timestamp ops:
     utimensat, futimens, futimes, utimes
   - *at variants (POSIX 2008 dirfd-based):
     faccessat, fchmodat, fchownat, renameat, readlinkat,
     symlinkat, mkdirat
   - Memory advisory: posix_fadvise, posix_fallocate
   - sendfile (Linux zero-copy)
   - Forks/exec: vfork, execvpe, execle, execlp, execl, execv,
     daemon
   - eventfd / timerfd_create / signalfd (Linux extensions)

   None of these are implemented yet on TaterTOS at the libc level
   — but they don't need to be for LibCore.a to archive. The .a
   archive doesn't link; symbol resolution only matters when an
   executable links against it. We provide implementations as
   subsequent ports demand each one.

B. unistd.h: include sys/types.h
   For dev_t, ino_t, mode_t, time_t (used in the new declarations).
   _SC_PHYS_PAGES, _SC_AVPHYS_PAGES, _SC_NPROCESSORS_CONF,
   _SC_LINE_MAX, _SC_HOST_NAME_MAX added (sysconf names AK uses).

C. unistd.h: ftruncate / dup2 / readlink / symlink / getcwd
   inline forwarders to fry_* / *_compat primitives.

D. unistd.h: signalfd type alias workaround
   Originally took `const sigset_t *mask`. sigset_t lives in
   <signal.h> which I didn't want to pull from <unistd.h> (cycle
   risk). Used a typedef alias _tatertos_sigset_alias_t
   (uint64_t — same width as our sigset_t). Acceptable because
   signalfd is never called by AK/LibCore at the source level —
   only the declaration needs to parse.
