Bucket C — TARGET_OS GATED (need a TaterTOS branch).
=====================================================================
These have explicit #[cfg(target_os = "linux")] / "macos" /
"windows" / "wasi" branches. We add #[cfg(target_os = "tatertos")]
forks and ship a vendored copy under our control.

  libc          — Rust's POSIX surface. Add tatertos branch
                  pointing at TaterTOS's libc.h surface (we
                  already shipped Chunks 1-2 with the equivalents).
                  Effort: medium. ~200-500 LOC of typedef +
                  function-decl mirrors.

  getrandom     — entropy source. Single-function add: route to
                  fry_getrandom (already wired through cxx_runtime
                  for AK::Random in Chunk 2). Effort: small.
                  ~30 LOC.

  rustix        — POSIX syscall wrapper. Note: rustix is in
                  Bucket A (host-only via tempfile). For TaterTOS
                  target it's NOT linked. Skip.

  r-efi         — UEFI helpers. Not used by TaterTOS userland.
                  Skip on the target.

  once_cell     — lazy initialisation. Has Linux/Win/macOS
                  threading branches but a generic spinlock
                  fallback exists; either pick fallback or add
                  tatertos branch using fry_futex. Effort: small.

  once_cell_polyfill — backport of once_cell from std. Same
                       considerations.

  anyhow        — error handling. Should be portable; check for
                  std::backtrace deps that may need patching.

  Subtotal: ~7 crates POSSIBLY NEED PATCHING.
