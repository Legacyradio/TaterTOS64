OBJECTIVE
=====================================================================
Enable full networking capabilities for the Ladybird port on TaterTOS
by integrating LibCore socket abstractions with TaterTOS kernel
networking syscalls. This is an honest, no-shortcuts path to
functional web connectivity.

CURRENT STATE
=====================================================================
- Kernel: Supports BSD-like sockets (TCP/UDP) via fry_ syscalls.
- LibC: src/user/libc/posix.c has basic wrappers for socket, bind,
  connect, listen, accept, send, recv, poll.
- DNS: src/user/libc/netdb.c implements getaddrinfo using the
  fry_dns_resolve syscall.
- Event Loop: EventLoopImplementationTaterTOS is functional and uses
  poll(), which is compatible with socket FDs.
- verified: posix.c has socket(), bind(), connect(), listen(),
  accept(), send(), recv(), sendto(), recvfrom(), poll().
- identified: accept4() is a shallow wrapper around accept(), ignoring
  flags like SOCK_NONBLOCK and SOCK_CLOEXEC.

§1 - AUDIT & ENHANCE POSIX COMPATIBILITY
=====================================================================
1.1  Update src/user/libc/posix.c:accept4()
     - Currently ignores flags. Must handle SOCK_NONBLOCK and
       SOCK_CLOEXEC.
     - If kernel fry_accept doesn't support flags, use fcntl(F_SETFL,
       O_NONBLOCK) and fcntl(F_SETFD, FD_CLOEXEC) manually on the
       new FD.
     - Log: fry856

1.2  Verify/Add Socket Options
     - Ensure getsockopt() and setsockopt() in posix.c handle
       SO_REUSEADDR, SO_ERROR, TCP_NODELAY.
     - Map these to fry_setsockopt/fry_getsockopt.
     - Log: fry857

1.3  Error Mapping Audit
     - Verify that all networking syscalls in posix.c correctly set
       errno from the negative return value of fry_ syscalls.
     - Log: fry858

§2 - LIBCORE SOCKET INTEGRATION
=====================================================================
2.1  Verify src/user/apps/ladybird/Libraries/LibCore/System.cpp
     - Audit wrappers for socket, bind, connect, etc.
     - Ensure they are calling the TaterTOS LibC posix symbols.
     - No "minimal" stubs; must return ErrorOr<T> correctly.
     - Log: fry859

2.2  Audit src/user/apps/ladybird/Libraries/LibCore/Socket.cpp
     - Ensure Socket::resolve_host uses the TaterTOS getaddrinfo.

     - Log: fry860

2.3  Local Sockets (AF_LOCAL / AF_UNIX)
