HOST TOOLCHAIN
=====================================================================
The upstream Host_Tools preset expected a missing vcpkg checkout under
ladybird-upstream/Build/vcpkg, so a local host-tools CMake build was
configured directly:

  cmake -S /home/legacyindieradio/ladybird-upstream \
        -B /home/legacyindieradio/ladybird-upstream/Build/tatertos-host-tools \
        -G Ninja \
        -DCMAKE_MAKE_PROGRAM=/usr/bin/ninja \
        -DLAGOM_TOOLS_ONLY=ON \
        -DINSTALL_LAGOM_TOOLS=ON \
        -DBUILD_SHARED_LIBS=OFF \
        -DCMAKE_BUILD_TYPE=Release \
        -DENABLE_GUI_TARGETS=OFF \
        -DENABLE_LAGOM_CCACHE=OFF \
        -DENABLE_NETWORK_DOWNLOADS=OFF \
        -DENABLE_INSTALL_HEADERS=OFF \
        -DLADYBIRD_ENABLE_CPPTRACE=OFF \
        -DTATERTOS_LADYBIRD_3RD=/home/legacyindieradio/TaterTOS64/src/user/apps/ladybird-3rd

Host tools built:

  cmake --build /home/legacyindieradio/ladybird-upstream/Build/tatertos-host-tools \
        --target BindingsGenerator GenerateWindowOrWorkerInterfaces GenerateNamedCharacterReferences -j 4

This built host-compatible AK, LibCore, LibFileSystem, LibIDL, LibMain,
and the required generator binaries.
