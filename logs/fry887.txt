What Changed
- Added public <sys/uio.h> with struct iovec plus readv/writev declarations.
- Added private fry_iovec ABI structure and fry_readv/fry_writev/fry_openat declarations.
- Added libc syscall numbers:
  - SYS_OPENAT = 105
  - SYS_READV = 106
  - SYS_WRITEV = 107
- Added fry_openat, fry_readv, and fry_writev inline syscall wrappers.
- Added POSIX openat, readv, and writev wrappers with errno translation.
- Added kernel SYS_OPENAT handler for absolute paths and AT_FDCWD-relative paths.
- Added kernel SYS_READV and SYS_WRITEV handlers using existing SYS_READ/SYS_WRITE paths.
- Enforced iovcnt bounds, user iovec validation, buffer access checks, total length overflow checks, and partial I/O semantics.

Why
- Chromium and GN-built userspace expect POSIX openat/readv/writev APIs early in the port.
- Implementing these as real syscall ABI entries keeps behavior available to all future C/C++ ports instead of adding one-off libc stubs.
- Returning ENOTSUP for directory-fd-relative openat paths avoids faking unsupported kernel cwd/dirfd state.
