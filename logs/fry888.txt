What Changed
- Added public POSIX msg/cmsg socket declarations:
  - struct msghdr
  - struct cmsghdr
  - CMSG_* helper macros
  - sendmsg()/recvmsg() declarations
  - MSG_TRUNC and MSG_CTRUNC
- Added private fry_msghdr ABI structure.
- Added libc syscall numbers:
  - SYS_SENDMSG = 108
  - SYS_RECVMSG = 109
- Added fry_sendmsg and fry_recvmsg syscall wrappers.
- Replaced the old userspace sendmsg/recvmsg hard -1 stubs with errno-translating syscall wrappers.
- Added kernel SYS_SENDMSG handler:
  - validates msghdr and iovec arrays
  - gathers iovec data into a kernel linear buffer
  - supports TCP stream sends
  - supports UDP datagram sends with optional msg_name destination
  - rejects unsupported outgoing control messages with ENOTSUP
  - enforces UDP payload sizing with EMSGSIZE
- Added kernel SYS_RECVMSG handler:
  - validates writable msghdr and destination iovec arrays
  - scatters received data into user iovecs
  - supports TCP stream receives
  - supports UDP datagram receives with optional source msg_name writeback
  - sets MSG_TRUNC for truncated UDP datagrams
  - reports absent ancillary data by zeroing msg_controllen

Why
- Chromium networking code expects sendmsg/recvmsg and msghdr/cmsg declarations.
- Implementing these as real kernel syscalls keeps behavior available to all ports and avoids libc-only shortcuts.
- Explicit ENOTSUP for unsupported send control data prevents silently lying about SCM_RIGHTS/ancillary semantics.
