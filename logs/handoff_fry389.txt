SESSION SUMMARY
---------------
Battery shows "unavailable" and backlight shows 0% on Dell Precision 7530
bare metal despite brightness cranked in BIOS. No serial cable available,
so all kprint() diagnostics during ACPI init are invisible. Added a new
SYS_ACPI_DIAG syscall (28) and expanded the sysinfo TaterWin app to
display ACPI discovery status on-screen.

PROBLEM
-------
After fry379 (EC driver) and fry388 (ACPI namespace hardening), battery
and backlight are still non-functional on real Dell hardware. The kernel
discovery code logs diagnostic info via kprint() → COM1 serial, but on
bare metal there is no serial output. We need on-screen visibility to
determine WHERE the pipeline breaks:
  1. EC not responding?
  2. Namespace walk not finding PNP0C0A (battery) or ACPI0008 (backlight)?
  3. Nodes found but _BST/_BQC returning zeros?

INTERPRETATION GUIDE
--------------------
- EC=FAIL → EC not responding on either port pair; need different init
- Batt found=0 → ns_walk didn't find PNP0C0A HID or _BST+_BIF/_BIX combo
  → namespace parsing issue or Dell uses non-standard battery device
- Batt found>0, Battery still "unavailable" → _BST eval returning bad pkg
  → AML execution or EC OpRegion read issue
- BL found=0 → no ACPI0008 HID or _BCM/_BQC/_BCL methods in namespace
  → Dell may use vendor-specific backlight (Intel IGFX OpRegion?)
- BL found=1, pct=0, bcl=0 → node found but no _BCL package
- BL found=1, bcl>0, raw=0 → _BQC returning 0 despite HW brightness set
