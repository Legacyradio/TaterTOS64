SESSION SUMMARY
========================================================================
Fixed EC initialization deadlock on Dell Precision 7530. The EC was
reporting FAIL because ec_reg_init() probed the EC BEFORE calling
_REG(3,1). Dell ECs won't respond to direct port I/O until _REG
announces a region handler. But _REG was only called after a successful
probe — creating a chicken-and-egg deadlock where neither ever happens.

PROBLEM
-------
Sysinfo showed: EC: FAIL ports=0x62/0x66
Battery: STA=0x00000000 (because battery _STA reads EC registers)

Old flow:
  1. Discover ports (ECDT / _CRS / defaults)
  2. ec_init_ports() → probe EC 5 times → ALL FAIL → return early
  3. _REG(3,1) → NEVER REACHED

FIX APPLIED
-----------
New flow:
  1. Discover ports (ECDT / _CRS / defaults) — unchanged
  2. ec_setup_ports() — sets ports, enables ACPI (SCI_EN), sets
     g_ec_ok=1 optimistically (so ec_read/ec_write work during _REG)
  3. EC _INI method called if it exists
  4. _REG(3,1) called — tells AML that EC region handler is active
  5. EC _GPE enabled via acpi_enable_gpe() — wakes EC from idle
  6. ec_probe() — NOW probe with 5 retries + alternate port fallback

Files changed:
  src/kernel/acpi/ec.c         — added ec_setup_ports(), ec_probe()
  src/kernel/acpi/ec.h         — exported new functions
  src/kernel/acpi/extended.c   — restructured ec_reg_init(), added events.h
