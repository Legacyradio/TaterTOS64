ROOT CAUSE IDENTIFIED: PCR[DMI] Sideband Mirror Missing
---------------------------------------------------------
On Intel 300 series PCH (Coffee Lake, which Dell Precision 7530 uses),
the PCI config space IOE register at offset 0x82 is NOT the authoritative
decode control. The ACTUAL I/O decode is governed by PCR[DMI] sideband
registers accessed through the P2SB (Primary to Sideband Bridge).

Evidence from coreboot (reference only — NOT copied into OS):
  - coreboot's lpc_enable_fixed_io_ranges() explicitly mirrors IOE to
    GPMR/PCR[DMI] via gpmr_write32(GPMR_LPCIOE, io_enables)
  - Without this mirror, the eSPI controller may not forward I/O cycles
    to the EC even though PCI config 0x82 looks correct
  - This explains: PCI config IOE=0x3F0F (bit 11 set = EC 0x62/0x66),
    PS/2 at 0x60/0x64 works (KBC bit mirrored by Dell BIOS),
    but EC at 0x62/0x66 returns 0xFF (EC bit NOT mirrored by Dell BIOS)

Key register addresses (from Intel PCH datasheets & coreboot headers):
  PID_DMI           = 0x88       (DMI port ID in PCR space)
  PCR_DMI_LPCIOD    = 0x2770     (LPC I/O Decode Ranges mirror)
  PCR_DMI_LPCIOE    = 0x2774     (LPC I/O Enable mirror)
  SBREG_BAR default = 0xFD000000 (P2SB sideband register base)
  P2SB device       = PCI 00:1f.1 (hidden by Dell BIOS)
  P2SB hide bit     = config offset 0xE0, bit 8 (byte 0xE1 bit 0)

  PCR physical addr = SBREG_BAR + (PID << 16) + offset
  DMI LPCIOE phys   = 0xFD000000 + (0x88 << 16) + 0x2774 = 0xFD882774

IOE bit definitions (from coreboot lpc_lib.h — reference only):
  LPC_IOE_COMA_EN       = (1 << 0)   // COM A 3F8-3FF
  LPC_IOE_COMB_EN       = (1 << 1)   // COM B 2F8-2FF
  LPC_IOE_LPT_EN        = (1 << 2)   // LPT 378-37F
  LPC_IOE_FDD_EN        = (1 << 3)   // FDD 3F0-3F7
  LPC_IOE_LGE_200       = (1 << 8)   // Game 200-207
  LPC_IOE_HGE_208       = (1 << 9)   // Game 208-20F
  LPC_IOE_KBC_60_64     = (1 << 10)  // KBC (PS/2) 60h/64h
  LPC_IOE_EC_62_66      = (1 << 11)  // EC 62h/66h  <<< THIS IS THE KEY BIT
  LPC_IOE_SUPERIO_2E_2F = (1 << 12)  // SuperIO 2Eh/2Fh
  LPC_IOE_EC_4E_4F      = (1 << 13)  // EC alt 4Eh/4Fh

EXISTING BUG FOUND:
  Line 619 of extended.c says:
    ioe_after = ioe_before | (1u << 13);  /* MC_LPC2_EN: decode 0x62/0x66 */
  This is WRONG. Bit 13 = EC_4E_4F (decode 0x4E/0x4F), NOT 0x62/0x66.
  The correct bit for EC 0x62/0x66 is bit 11.
  It was harmless because Dell BIOS sets 0x3F0F (all bits already set),
  but the code and comment are incorrect and must be fixed.

========================================================================
CROSS-COMPATIBILITY NOTES
========================================================================
- PCR mirror only runs on Intel (vendor 0x8086) ISA bridges (class 06/01)
- AMD systems: ec_enable_lpc_decode() skips writes (vendor != 0x8086)
- Older Intel (pre-Skylake): PCR mirror is harmless — same register layout
  existed from Skylake onwards; older systems don't use eSPI so PCI config
  IOE alone suffices, and writing PCR is a no-op if SBREG isn't configured
- Non-Intel P2SB: pcr_get_sbreg_bar() checks vendor == 0x8086 before
  touching P2SB, falls back to SBREG_BAR_DEFAULT which is safe
