WHAT WAS FIXED THIS SESSION
---------------------------
1) Bare-metal ACPI EC fallback robustness in `src/kernel/acpi/extended.c`
   - Replaced strict yes/no PCR plausibility filtering with score-based
     candidate selection for SBREG BAR + PID.
   - Added alternate IOD hint usage from LPC register 0x88 low word.
   - Result: Dell-valid register patterns are less likely to be discarded
     when P2SB unhide fails and fallback probing is required.

2) DBG_VFS/UFS noise cleanup in ramdisk path
   - `src/kernel/fs/vfs.c`: `rd_readdir()` now rejects non-root paths for
     flat ramdisk layout instead of replaying root entries repeatedly.
   - `src/kernel/main.c`: changed debug prefix text from
     `ERROR: DBG_VFS ...` to `DBG_VFS ...` for diagnostics.

WHY THESE CHANGES
-----------------
- User-reported logs showed repeated debug entry spam and EC probe failures on
  Dell bare metal. Previous fallback heuristics could reject plausible PCR
  candidates too aggressively. The scoring-based path is more tolerant while
  still rejecting all-FF/all-00 invalid reads.
