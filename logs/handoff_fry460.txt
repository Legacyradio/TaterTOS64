IMPLEMENTED CHANGES
-------------------
1) New graphical app: FILEMAN.FRY
   - File: src/user/apps/fileman.c
   - TaterWin protocol implemented:
     * TW_MSG_CREATE_WINDOW request
     * SHM mapping via WINDOW_CREATED/RESIZED
     * redraw + TW_MSG_UPDATE loop
   - Uses existing libc syscalls only (no new syscall added):
     * fry_readdir, fry_stat, fry_spawn, fry_storage_info, fry_read
   - UI/layout:
     * top path bar
     * left "Places"
     * right file list (dirs + files, size/type)
     * status bar with root/secondary/nvme summary
   - Navigation:
     * j/k (or s/w) move selection
     * Enter/l open dir or launch .FRY
     * h/backspace go parent
     * 1..9 jump to place
     * r refresh
     * q quit

2) GUI app listing is now runtime-discovered (not fixed hardcoded list)
   - File: src/user/gui/gui.c
   - Start menu now scans `/` and `/fry` for `*.FRY` at runtime.
   - Launch fallback path order:
     * /APP.FRY
     * /fry/APP.FRY
   - Effect: removing a `.FRY` file removes it from menu on next open.

3) Build + packaging integration for FILEMAN
   - Files:
     * Makefile
     * build_iso.sh
   - Added target + packaging for:
     * fileman.elf
     * fileman.fry
     * FILEMAN.FRY copied into root and /fry for FAT/ISO/NVMe images

4) Multi-partition secondary mount support on same NVMe drive
   - File: src/kernel/fs/vfs.c
   - `vfs_mount_secondary()` now scans GPT and mounts multiple supported
     partitions, not just first match.
   - Mount scheme:
     * primary preferred secondary: /nvme
       (priority: ToTFS > NTFS > best FAT32)
     * additional supported partitions: /nvme1, /nvme2, ...
   - Supported secondary fs mount types:
     * ToTFS (rw)
     * NTFS (ro stubs for write/create/mkdir/unlink)
     * FAT32 (rw via existing ops where supported)

5) FAT32 mountpoint root stat fix
   - File: src/kernel/fs/vfs.c
   - `fat32_stat_vfs()` now returns directory attr for empty/relative-root
     path so mountpoints like `/nvmeX` are recognized as dirs in userspace.
