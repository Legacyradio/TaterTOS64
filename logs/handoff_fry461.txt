ROOT CAUSE FOUND
----------------
Primary issue causing "desktop freaks out/glitch refresh" behavior:
1) Start menu app discovery was including GUI.FRY as a normal launchable app.
   Launching that entry spawned a second desktop compositor instance.
   Result: two GUI processes contend for framebuffer updates, causing
   desktop-wide visual instability/flicker.

Secondary stability issue fixed in same pass:
2) fileman.c freed previous SHM IDs on resize from userspace.
   SHM lifecycle is owned by GUI window manager path; app-side free could
   invalidate buffers unexpectedly.

Additional hardening in this pass:
3) File Manager initial client size reduced from 920x560 to 760x460 to avoid
   oversized startup requests on lower-resolution GOP setups.

IMPLEMENTED CHANGES
-------------------
1) GUI start menu discovery/launch safeguards
   File: src/user/gui/gui.c
   - In app discovery, if an entry resolves to "GUI", remap menu name to
     "FILEMAN" so GUI compositor is not launchable as a user app.
   - Added FILEMAN to fallback menu set when directory listing is unavailable.
   - In launcher path, guard against "GUI" by remapping to "FILEMAN".

2) File Manager SHM handling + naming
   File: src/user/apps/fileman.c
   - Removed app-side `fry_shm_free(g_shm_id)` during resize handling.
   - Changed initial size:
     * INIT_W: 920 -> 760
     * INIT_H: 560 -> 460
   - Renamed visible/app title text:
     * "File Explorer" -> "File Manager"
