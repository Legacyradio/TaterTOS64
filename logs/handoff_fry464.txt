WHAT WAS DONE (fry464)
----------------------
FILEMAN.FRY was missing from the EFI loader's hardcoded ramdisk file
list in src/boot/efi_loader.c. The loader only packed 5 files:
  GUI.FRY, SHELL.TOT, SYSINFO.FRY, UPTIME.FRY, PS.FRY

On Dell ISO boot, the ramdisk contained only those 5. The GUI's
discover_apps() scanned "/" via fry_readdir, found those 5, so
app_count > 0 and the fallback list (which did include FILEMAN) never
ran. Result: FILEMAN never appeared in the start menu.

Fix: bumped FRY_FILE_COUNT from 5 to 6, added FILEMAN.FRY path
variants (8 paths) and "FILEMAN.FRY" to fry_names[].

Also in fry463 (previous session): gui.c filters out "GUI" from the
start menu (line 156) so launching a second compositor is impossible.
