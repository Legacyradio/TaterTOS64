       874 +  13a. AMD FCH LPC bridge detection
       875 +    - Scan PCI for vendor=0x1022, class=06/01 (ISA bridge)
       876 +    - Common AMD LPC bridge device IDs:
       877 +        0x790E (Ryzen / Zen FCH), 0x780E (Kabini/Mullins),
       878 +        0x439D (Hudson/Bolton), 0x7901 (older FCH variants)
       879 +    - If found: AMD FCH path. If not: skip (Intel or unknown).
       880 +
       881 +  13b. AMD FCH I/O decode enable
       882 +    - AMD FCH does NOT use Intel's P2SB/PCR sideband architecture.
       883 +    - EC 0x62/0x66 decode on AMD FCH:
       884 +        Option A: Wide I/O decode range registers at LPC PCI config
       885 +                  offsets 0x64/0x66 (WideIO 1) and 0x90 (WideIO 2).
       886 +                  Set base=0x0062, size=4 bytes → decode 0x62-0x65.
       887 +        Option B: Some AMD FCH variants decode 0x60-0x67 by default
       888 +                  when KBC decode is enabled (LPC config offset 0x48
       889 +                  bit 2 = Decode_KBC_IO, covers 0x60/0x64 + 0x62/0x66).
       890 +    - Strategy: read 0x48, check/set bit 2 (KBC_IO). If EC still
       891 +      unreachable, program WideIO to cover 0x62-0x66.
       892 +    - Gated by vendor=0x1022 check. Intel path unchanged.
       893 +
       894 +  13c. AMD ACPI EC differences
       895 +    - ACPI EC interface (_CRS ports, _REG, _GPE) is identical on AMD.
       896 +    - PNP0C09 namespace node, EmbeddedControl region handler same.
       897 +    - Only the LPC decode setup differs (FCH registers vs PCR[DMI]).
       898 +    - ec_enable_lpc_decode() already has vendor gating — extend it:
       899 +        if vendor==0x8086: Intel PCR[DMI] path (existing)
       900 +        if vendor==0x1022: AMD FCH path (new)
       901 +        else: skip (log warning, rely on firmware decode)

PHASE 18 - WRITABLE FILESYSTEM
=====================================================================
FAT32 is currently read-only. This phase adds full read-write support
so the OS can create, modify, and delete files on NVMe.

kernel/fs/fat32_write.c (or extend fat32.c):
  fat32_alloc_cluster():
    Read FSInfo sector (sector 1 of partition, or fsinfo_sector from BPB).
    FSInfo struct: signature 0x41615252, free_count, next_free hint.
    Starting at next_free hint (or cluster 2 if hint invalid):
      Scan FAT entries for value 0x00000000 (free cluster).
    Mark found cluster as 0x0FFFFFFF (end-of-chain) in FAT.
    Update both FAT copies (FAT1 and FAT2 if num_fats == 2).
    Decrement FSInfo.free_count. Update FSInfo.next_free.
    Write FSInfo sector back to disk.
    Return allocated cluster number.
  fat32_extend_chain(last_cluster, new_cluster):
    Write new_cluster into FAT entry for last_cluster.
    Write 0x0FFFFFFF into FAT entry for new_cluster.
    Update both FAT copies.
  fat32_free_chain(start_cluster):
    Walk chain from start_cluster. For each cluster:
      Read FAT entry to get next.
      Write 0x00000000 to FAT entry (mark free).
    Update FSInfo.free_count += freed count.
    Write FSInfo sector.
  fat32_write_file(file, buf, len):
    If file has no clusters yet: allocate first cluster, update dir entry.
    Write data to current cluster's sectors.
    If data exceeds cluster boundary:
      Allocate new cluster via fat32_alloc_cluster().
      Chain via fat32_extend_chain().
      Continue writing to new cluster.
    Update dir entry: file_size, write_time, write_date.
    Flush modified FAT sectors and directory sector to disk.
  fat32_create(dir_cluster, name, is_dir):
    Find free 32-byte slot in directory cluster chain.
    If no free slot: extend directory chain (allocate cluster).
    Generate 8.3 name from long name:
      Uppercase, truncate to 8+3, add ~N suffix if collision.
    If name > 11 chars or has lowercase: create LFN entries.
      LFN entries precede 8.3 entry in reverse sequence order.
      Compute checksum from 8.3 name.
    Fill directory entry:
      name[11], attr (0x10 for dir, 0x20 for file),
      create_time, create_date, write_time, write_date,
      first_cluster_hi (bytes 20-21), first_cluster_lo (bytes 26-27),
      file_size (0 for dir, 0 for new file).
    If is_dir:
      Allocate one cluster for the new directory.
      Create "." entry (points to self cluster).
      Create ".." entry (points to parent dir_cluster).
      Zero-fill rest of cluster.
    Write directory sector(s) back to disk.
    Return 0 on success.
  fat32_delete(dir_cluster, name):
    Find entry in directory. Mark first byte as 0xE5 (deleted).
    If LFN entries exist: mark each LFN entry byte 0 as 0xE5.
    Free the file's cluster chain via fat32_free_chain().
    Write directory sector back.
  fat32_rename(old_dir, old_name, new_dir, new_name):
    Read old entry. Create new entry in new_dir.
    Copy cluster chain pointer (don't move data).
    Delete old entry (mark 0xE5, don't free clusters).
  fat32_truncate(file, new_size):
    If new_size < current size:
      Calculate which cluster offset corresponds to new_size.
      Free all clusters beyond that point.
      Update FAT: mark last-kept cluster as 0x0FFFFFFF.
    Update dir entry file_size = new_size.
  fat32_sync():
    Flush all dirty FAT sectors cached in memory to NVMe.
    Flush all dirty directory sectors.
    Update FSInfo sector.
    This is called on unmount and periodically.

PHASE 18a - READ-ONLY NTFS SUPPORT
=====================================================================
Read-only NTFS driver for mounting Windows partitions (Dell Precision
7530 ships with NTFS on its NVMe). Enables browsing/reading Windows
files from TaterTOS64v3.

Goal: Auto-detect NTFS partitions on NVMe at boot, mount alongside
ramdisk at /nvme, allow ls/cd/cat of NTFS files from shell.

src/kernel/fs/ntfs.h:
  On-disk structures (all packed, little-endian):
    ntfs_boot_sector: OEM ID "NTFS    ", bytes/sector, sectors/cluster,
      MFT start LCN, MFT mirror LCN, clusters per MFT record, serial.
    ntfs_mft_entry: magic "FILE" (0x454C4946), fixup array offset/count,
      sequence number, hard link count, first attr offset, flags (in-use/dir).
    ntfs_attr_header: type (0x10/$STANDARD_INFO, 0x30/$FILE_NAME,
      0x80/$DATA, 0x90/$INDEX_ROOT, 0xA0/$INDEX_ALLOCATION, 0xB0/$BITMAP),
      length, non-resident flag, name length/offset.
    ntfs_attr_resident: value offset, value length.
    ntfs_attr_nonresident: start VCN, end VCN, data run offset,
      allocated size, real size, initialized size.
    ntfs_filename_attr: parent MFT ref, name length, namespace, UTF-16LE name.
    ntfs_index_header / ntfs_index_entry: directory B-tree structures.
  Constants:
    NTFS_MAGIC = 0x454C4946 ("FILE")
    NTFS_ATTR_STANDARD_INFO=0x10, NTFS_ATTR_FILE_NAME=0x30,
    NTFS_ATTR_DATA=0x80, NTFS_ATTR_INDEX_ROOT=0x90,
    NTFS_ATTR_INDEX_ALLOC=0xA0, NTFS_ATTR_BITMAP=0xB0
    NTFS_ATTR_END=0xFFFFFFFF
    NTFS_FILE_ROOT=5 (root directory MFT entry)
  Kernel structs:
    ntfs_fs: block device, partition LBA, bytes/sector, sectors/cluster,
      cluster size, MFT LCN, MFT record size, sector buffer.
    ntfs_file_private: fits in vfs_file.private[128].
  Driver API:
    ntfs_probe(bd, part_lba) → 1 if NTFS, 0 if not
    ntfs_mount(fs, bd, part_lba) → 0 on success
    ntfs_open_vfs, ntfs_read_vfs, ntfs_close_vfs, ntfs_readdir_vfs, ntfs_stat_vfs

PHASE 19 - TCP/IP NETWORKING STACK
=====================================================================
Phase 12 provides raw Ethernet frames + ARP/ICMP only.
This phase builds a full TCP/IP stack for real network communication.

kernel/net/ip.c:
  IP layer (RFC 791):
  struct ip_header {
    uint8_t  ver_ihl;      // version (4) + IHL (5 = 20 bytes)
    uint8_t  tos;
    uint16_t total_len;
    uint16_t id;
    uint16_t flags_frag;   // DF, MF, fragment offset
    uint8_t  ttl;
    uint8_t  proto;        // 6=TCP, 17=UDP, 1=ICMP
    uint16_t checksum;
    uint32_t src_ip;
    uint32_t dst_ip;
  };
  ip_init():
    Set local_ip (from DHCP or static config).
    Set subnet_mask, gateway_ip, dns_ip.
  ip_send(dst_ip, proto, payload, len):
    Build IP header. Compute checksum (ones-complement sum).
    If dst_ip is on local subnet: ARP lookup dst_ip → MAC.
    Else: ARP lookup gateway_ip → MAC.
    Build Ethernet frame: dst_mac, src_mac, ethertype=0x0800.
    Call net_dev->send(frame, total_len).
    If payload > MTU (1500 - 20): fragment.
      Split into fragments with MF flag, fragment offset.
      Send each as separate IP packet.
  ip_receive(buf, len) (called from netcore):
    Validate: ver==4, checksum, total_len <= buf.
    If fragment: reassemble (store in fragment table by id+src_ip).
      When all fragments received: deliver complete datagram.
    Dispatch by proto:
      1 → icmp_receive() (already exists from Phase 12, extend)
      6 → tcp_receive()
      17 → udp_receive()

kernel/net/tcp.c:
  TCP layer (RFC 793 + RFC 5681 congestion control):
  struct tcp_header {
    uint16_t src_port, dst_port;
    uint32_t seq_num;
    uint32_t ack_num;
    uint8_t  data_offset;  // header length in 32-bit words
    uint8_t  flags;        // FIN, SYN, RST, PSH, ACK, URG
    uint16_t window;
    uint16_t checksum;
    uint16_t urgent_ptr;
  };
  TCP flags: FIN=0x01, SYN=0x02, RST=0x04, PSH=0x08, ACK=0x10, URG=0x20.
  struct tcp_conn {
    uint32_t local_ip, remote_ip;
    uint16_t local_port, remote_port;
    uint32_t snd_una;      // oldest unacked seq
    uint32_t snd_nxt;      // next seq to send
    uint32_t rcv_nxt;      // next expected seq from remote
    uint16_t snd_wnd;      // send window
    uint16_t rcv_wnd;      // receive window
    uint8_t  state;        // CLOSED, LISTEN, SYN_SENT, SYN_RCVD,
                           // ESTABLISHED, FIN_WAIT_1, FIN_WAIT_2,
                           // CLOSE_WAIT, CLOSING, LAST_ACK, TIME_WAIT
    uint8_t  *send_buf;    // retransmit buffer (circular, 64KB)
    uint8_t  *recv_buf;    // receive buffer (circular, 64KB)
    uint32_t send_buf_head, send_buf_tail;
    uint32_t recv_buf_head, recv_buf_tail;
    uint64_t rto_ms;       // retransmit timeout (initial 1000ms)
    uint64_t srtt_ms;      // smoothed RTT
    uint32_t cwnd;         // congestion window (bytes)
    uint32_t ssthresh;     // slow-start threshold
  };
  #define TCP_MAX_CONNS 64
  static struct tcp_conn conns[TCP_MAX_CONNS];
  tcp_connect(local_port, remote_ip, remote_port) → conn*:
    Allocate conn. Set state = SYN_SENT.
    Pick ISN (initial sequence number) from kernel timer + random.
    Send SYN packet (seq=ISN, ack=0, flags=SYN).
    Start retransmit timer. Wait for SYN+ACK.
  tcp_listen(local_port) → conn*:
    Allocate conn. Set state = LISTEN.
  tcp_accept(listen_conn) → conn*:
    Wait for SYN on listen port.
    Allocate new conn for accepted connection.
    Send SYN+ACK. Set state = SYN_RCVD.
    Wait for ACK → state = ESTABLISHED.
  tcp_send(conn, data, len):
    Copy data to send_buf. Segment into MSS-sized chunks (MSS = 1460).
    For each segment: build TCP header, compute checksum (pseudo-header +
    TCP header + payload), call ip_send().
    Advance snd_nxt. Start retransmit timer for each segment.
  tcp_receive(ip_src, buf, len):
    Parse TCP header. Find matching conn by (src_ip, src_port, dst_ip, dst_port).
    State machine:
      LISTEN + SYN: create conn, send SYN+ACK, → SYN_RCVD.
      SYN_SENT + SYN+ACK: send ACK, → ESTABLISHED.
      SYN_RCVD + ACK: → ESTABLISHED.
      ESTABLISHED + data: copy to recv_buf, send ACK, advance rcv_nxt.
      ESTABLISHED + FIN: send ACK, → CLOSE_WAIT.
      FIN_WAIT_1 + ACK: → FIN_WAIT_2.
      FIN_WAIT_2 + FIN: send ACK, → TIME_WAIT.
      (full state machine per RFC 793)
    Retransmit: if ACK received, slide snd_una forward, cancel retransmit
    timer for acked segments. If retransmit timer fires: resend from snd_una.
    Congestion control (RFC 5681):
      Slow start: cwnd += MSS on each ACK (while cwnd < ssthresh).
      Congestion avoidance: cwnd += MSS*MSS/cwnd on each ACK.
      Fast retransmit: 3 duplicate ACKs → retransmit, ssthresh=cwnd/2.
  tcp_close(conn):
    Send FIN. → FIN_WAIT_1.
    Wait for ACK+FIN. Send ACK. → TIME_WAIT.
    After 2*MSL (60s): free conn.
  tcp_read(conn, buf, len):
    Copy from recv_buf. Block (fry_sleep loop) if empty.
    Return bytes read.

kernel/net/udp.c:
  UDP layer (RFC 768):
  struct udp_header {
    uint16_t src_port, dst_port;
    uint16_t length;       // header (8) + payload
    uint16_t checksum;
  };
  struct udp_socket {
    uint16_t local_port;
    uint32_t local_ip;
    uint8_t  recv_buf[8192];
    uint32_t recv_head, recv_tail;
  };
  #define UDP_MAX_SOCKETS 32
  udp_bind(port) → socket*: allocate socket, bind to port.
  udp_sendto(sock, dst_ip, dst_port, data, len):
    Build UDP header. Compute checksum (pseudo-header).
    Call ip_send(dst_ip, 17, udp_packet, 8 + len).
  udp_receive(ip_src, buf, len):
    Parse header. Find socket by dst_port.
    Copy payload to socket recv_buf.
  udp_recvfrom(sock, buf, len, src_ip_out, src_port_out):
    Block (fry_sleep loop) until data in recv_buf.
    Copy out, return length + source info.

kernel/net/dns.c:
  DNS resolver (RFC 1035):
  dns_resolve(hostname) → uint32_t (IP address):
    Build DNS query packet:
      Header: random ID, QR=0, OPCODE=0, RD=1, QDCOUNT=1.
      Question: encode hostname as label sequence (length-prefixed segments).
      QTYPE=A (1), QCLASS=IN (1).
    Send via udp_sendto(dns_sock, dns_ip, 53, query, qlen).
    Wait response via udp_recvfrom (timeout 5s, retry up to 3 times).
    Parse response:
      Verify ID matches. Check RCODE (0 = no error).
      Skip question section.
      Walk answer RRs: find TYPE=A (1), CLASS=IN (1).
      Extract 4-byte RDATA = IP address.
    Cache result: simple array of {hostname, ip, ttl, timestamp}.
    Return IP.

kernel/net/dhcp.c:
  DHCP client (RFC 2131):
  dhcp_discover():
    Build DHCP DISCOVER packet:
      op=1 (BOOTREQUEST), htype=1 (Ethernet), hlen=6.
      xid=random transaction ID. flags=0x8000 (broadcast).
      chaddr=MAC address.
      Options: 53=DISCOVER, 55=parameter request list
        (subnet mask, router, DNS, lease time).
    Send via udp_sendto(broadcast 255.255.255.255, 67, packet, 300).
    Source port = 68 (DHCP client port).
  dhcp_offer_handler(buf, len):
    Parse OFFER: yiaddr (offered IP), siaddr (server IP).
    Options: 54=server identifier, 51=lease time, 1=subnet, 3=router, 6=DNS.
  dhcp_request(offered_ip, server_ip):
    Build DHCP REQUEST:
      Options: 53=REQUEST, 50=requested IP, 54=server ID.
    Send broadcast. Wait for ACK.
  dhcp_ack_handler(buf, len):
    Parse ACK: yiaddr confirmed.
    Apply: local_ip = yiaddr, subnet, gateway, dns from options.
    Start lease renewal timer (T1 = lease/2, T2 = lease*7/8).
  dhcp_init():
    Bind UDP socket on port 68.
    Call dhcp_discover(). Wait for OFFER (timeout 10s, retry 3x).
    Call dhcp_request(). Wait for ACK.
    On success: configure IP stack.
    On failure: fall back to link-local (169.254.x.x) or static.

  SYS_BIND(41):    fd, addr, addrlen → 0 or error.

  SYS_CONNECT(42): fd, addr, addrlen → 0 or error.

  SYS_SEND(45):    fd, buf, len, flags → bytes sent.

  SYS_RECV(46):    fd, buf, len, flags → bytes received.
    TCP: tcp_read(). UDP: udp_recvfrom().
  SYS_SENDTO(47):  fd, buf, len, flags, dest_addr → bytes sent.
    UDP: udp_sendto() with specified dest.
  SYS_RECVFROM(48): fd, buf, len, flags, src_addr_out → bytes received.
  SYS_CLOSE already handles socket cleanup (detect socket fd, free conn).

PHASE 20 - AUDIO SUBSYSTEM
=====================================================================
Intel HDA (High Definition Audio) is the standard audio controller
on the Dell Precision 7530 and virtually all x86 PCs.

kernel/drivers/audio/hda.c:
  PCI driver: class=0x04, subclass=0x03 (Audio Device).
  Intel HDA vendor=0x8086, various device IDs (Coffee Lake = 0xA348).

  HDA register map (BAR0, MMIO):
    0x00 GCAP    (Global Capabilities, 16-bit): num output/input streams.
    0x02 VMIN    (Minor Version).
    0x03 VMAJ    (Major Version).
    0x08 GCTL    (Global Control): CRST (controller reset), bit 0.
    0x0C WAKEEN  (Wake Enable).
    0x0E STATESTS (State Change Status): codec detection bits (bit N = codec N).
    0x10 GSTS    (Global Status).
    0x18 OUTPAY  (Output Payload Capability).
    0x1C INPAY   (Input Payload Capability).
    0x20 INTCTL  (Interrupt Control): GIE (global interrupt enable, bit 31),
         CIE (controller interrupt enable, bit 30), per-stream bits [0..29].
    0x24 INTSTS  (Interrupt Status).
    0x30 WALCLK  (Wall Clock Counter, 32-bit, 24MHz).
    0x38 SSYNC   (Stream Synchronization).
    0x40 CORBLBASE / 0x44 CORBUBASE (CORB ring buffer DMA address).
    0x48 CORBWP  (CORB Write Pointer).
    0x4A CORBRP  (CORB Read Pointer).
    0x4C CORBCTL (CORB Control): DMA run, MEIE.
    0x4E CORBSTS (CORB Status).
    0x50 CORBSIZE (CORB Size): 2/16/256 entries.
    0x50 RIRLBASE / 0x54 RIRBUBASE (RIRB ring buffer DMA address).
    0x58 RIRBWP  (RIRB Write Pointer).
    0x5A RINTCNT (Response Interrupt Count).
    0x5C RIRBCTL (RIRB Control): DMA run, interrupt enable.
    0x5E RIRBSTS (RIRB Status).
    0x60 RIRBSIZE (RIRB Size).
    0x70 DPLBASE / 0x74 DPUBASE (DMA Position in Buffer).
    0x80+ Stream descriptor registers (0x20 bytes per stream):
      +0x00 SD_CTL (Stream Descriptor Control): RUN, IOCE, FEIE, STRIPE.
      +0x03 SD_STS (Status): BCIS (buffer complete), FIFOE, DESE.
      +0x04 SD_LPIB (Link Position in Buffer).
      +0x08 SD_CBL (Cyclic Buffer Length, total bytes).
      +0x0C SD_LVI (Last Valid Index, BDL entry count - 1).
      +0x0E SD_FIFOS (FIFO Size).
      +0x10 SD_FMT (Stream Format):
        bits 15: type (0=PCM).
        bits 14: base rate (0=48kHz, 1=44.1kHz).
        bits 13-11: rate multiplier (x1, x2, x3, x4).
        bits 10-8: rate divisor (/1, /2, /3, /4, /5, /6, /7, /8).
        bits 7-4: bits per sample (000=8, 001=16, 010=20, 011=24, 100=32).
        bits 3-0: number of channels - 1.
      +0x18 SD_BDPL / +0x1C SD_BDPU (Buffer Descriptor List address).

  hda_init(pci_dev):
    Map BAR0 (typically 16KB).
    Enable bus master (PCI command register bit 2).
    Reset controller: clear GCTL.CRST, wait CRST=0, set CRST, wait CRST=1.
    Wait STATESTS to detect codecs (timeout 1s). Log which codec slots have devices.
    Allocate CORB ring (256 entries * 4 bytes = 1KB, DMA-coherent).
    Allocate RIRB ring (256 entries * 8 bytes = 2KB, DMA-coherent).
    Write CORBLBASE/CORBUBASE, RIRLBASE/RIRBUBASE.
    Set CORBSIZE=2 (256 entries), RIRBSIZE=2 (256 entries).
    Reset CORBRP (write bit 15, wait clear). Set CORBWP=0.
    Enable CORB DMA: CORBCTL.DMA_RUN=1.
    Enable RIRB DMA: RIRBCTL.DMA_RUN=1.
    Set RINTCNT=1 (interrupt after each response).
    Enable interrupts: INTCTL = GIE | CIE.
    Allocate MSI vector for HDA interrupts.

  hda_send_verb(codec_id, nid, verb) → uint32_t response:
    CORB entry format: bits [31:28]=codec, [27:20]=NID, [19:0]=verb.
    Write entry to CORB[next]. Advance CORBWP.
    Poll RIRBWP until response appears (timeout 1s).
    Read response from RIRB[rirb_read_ptr]: {response, response_ex}.
    Advance rirb_read_ptr. Return response.

  Codec verbs (HDA spec section 7.3):
    GET_PARAMETER (0xF00): subtypes:
      0x00 = vendor_id+device_id.
      0x02 = subordinate node count (start_nid, total).
      0x04 = function group type.
      0x05 = audio function group caps.
      0x09 = widget capabilities.
      0x0A = supported PCM sizes/rates.
      0x0B = supported stream formats.
      0x0D = output amplifier capabilities.
      0x0E = connection list length.
      0x0F = supported power states.
    GET_CONNECTION_SELECT (0xF01).
    SET_CONNECTION_SELECT (0x701).
    GET_CONNECTION_LIST_ENTRY (0xF02).
    SET_CONVERTER_FORMAT (0x200 + format).
    SET_CONVERTER_STREAM_CHANNEL (0x706): bits [7:4]=stream_id, [3:0]=channel.
    SET_PIN_WIDGET_CONTROL (0x707): OUT=0x40, IN=0x20, HP=0x80.
    GET_PIN_CONFIG_DEFAULT (0xF1C): device type, location, connector, color.
    SET_AMP_GAIN_MUTE (0x300):
      bit 15: output (1) or input (0).
      bit 13: left (1).
      bit 12: right (1).
      bit 7: mute.
      bits 6:0: gain (0-127).
    SET_POWER_STATE (0x705): D0=0, D1=1, D2=2, D3=3.

  hda_enumerate_codec(codec_id):
    Send GET_PARAMETER(root_nid=0, vendor_id). Log codec chip.
    Send GET_PARAMETER(root_nid=0, subordinate_node_count).
    For first function group (typically AFG, nid=1):
      Send GET_PARAMETER(nid=1, subordinate_node_count) → widget range.
      For each widget NID in range:
        GET_PARAMETER(nid, widget_caps): type field (bits 23:20):
          0=Audio Output, 1=Audio Input, 2=Audio Mixer,
          3=Audio Selector, 4=Pin Complex, 5=Power Widget.
        If Pin Complex: GET_PIN_CONFIG_DEFAULT to determine:
          Port connectivity (bits 31:30): 0=jack, 1=no phys, 2=fixed, 3=both.
          Default device (bits 23:20): 0=line out, 1=speaker, 2=HP out,
            3=CD, 4=SPDIF out, 5=digital, 6=modem, 8=line in,
            9=AUX, 0xA=mic in.
          Color, location, connector type.
        If Audio Output: record DAC NID, supported formats/rates.
        If Audio Mixer/Selector: record connection list.
    Build codec topology graph: PIN → MIXER → DAC chains.
    Find default output path: Speaker Pin → Mixer → DAC.
    Find headphone path: HP Pin → Mixer → DAC (or separate DAC).

  hda_setup_stream(stream_id, sample_rate, bits, channels):
    Compute SD_FMT value from parameters.
    Stop stream: clear SD_CTL.RUN, wait SD_CTL.RUN=0.
    Set SD_FMT. Set SD_CBL (buffer length).
    Allocate BDL (Buffer Descriptor List): array of {addr, len, IOC_flag}.
      Typical: 2 BDL entries for double-buffered playback.
      Each entry: 4KB-32KB depending on latency requirements.
    Write SD_BDPL/SD_BDPU with BDL physical address.
    Set SD_LVI (last valid index = num_bdl_entries - 1).
    Set converter on DAC: SET_CONVERTER_STREAM_CHANNEL(dac_nid, stream_id, 0).
    Set converter format: SET_CONVERTER_FORMAT(dac_nid, fmt).
    Unmute output path:
      SET_AMP_GAIN_MUTE on each widget in DAC→PIN chain.
      SET_PIN_WIDGET_CONTROL on output pin (OUT or HP enable).
    Start stream: SD_CTL.RUN=1, SD_CTL.IOCE=1 (interrupt on complete).

  hda_playback_fill(stream_id, pcm_data, len):
    Find current stream position from SD_LPIB (or DMA position buffer).
    Calculate which BDL buffer is safe to write (not currently being read).
    Copy PCM data to that buffer.
    If using double-buffer: alternate between buffer 0 and buffer 1.
    Interrupt handler: on BCIS (buffer complete), signal userspace to
    provide next chunk.

  hda_set_volume(dac_nid, left_gain, right_gain):
    SET_AMP_GAIN_MUTE(output, left, gain=left_gain).
    SET_AMP_GAIN_MUTE(output, right, gain=right_gain).
    SET_AMP_GAIN_MUTE(output, mute=0) to unmute.

  IRQ handler:
    Read INTSTS. For each stream with interrupt pending:
      Clear SD_STS.BCIS (write 1 to clear).
      Signal playback/capture callback.
    Read RIRBSTS for unsolicited responses.
    lapic_eoi().

kernel/proc/syscall.c audio syscalls:
  SYS_AUDIO_OPEN(50): sample_rate, bits, channels → audio_fd.
    Configures HDA output stream. Returns fd for PCM writes.
  SYS_AUDIO_WRITE(51): audio_fd, pcm_buf, len → bytes queued.
    Copies PCM data to stream DMA buffer.
  SYS_AUDIO_CLOSE(52): audio_fd → stops stream, frees resources.
  SYS_AUDIO_VOLUME(53): left, right → sets master volume (0-100).

user/libc/libc.c audio wrappers:
  int fry_audio_open(int rate, int bits, int channels);
  long fry_audio_write(int fd, const void *pcm, long len);
  void fry_audio_close(int fd);
  void fry_audio_volume(int left, int right);

src/user/libc/font.h and font.c:
  PSF2 font format loader (PC Screen Font, version 2):
    PSF2 header (32 bytes):
      magic: 0x864AB572.
      version: 0.
      headersize: 32.
      flags: 0x01 = has Unicode table.
      numglyph: number of glyphs (typically 256 or 512).
      bytesperglyph: height * ((width + 7) / 8).
      height: glyph height in pixels.
      width: glyph width in pixels.
    After header: raw bitmap data, 1 bit per pixel, row-major.
    After glyphs (if has Unicode table): Unicode mapping entries.
  font_load(data, len) → font_t*:
    Validate magic. Store header fields.
    Pointer to glyph data = data + headersize.
  font_draw_char(ctx, font, x, y, ch, fg, bg):
    Look up glyph index from ch (direct for ASCII, Unicode table for extended).
    For each row in glyph height:
      For each column in glyph width:
        If bit set: draw fg pixel at (x+col, y+row).
        Else if bg != transparent: draw bg pixel.
  font_draw_text(ctx, font, x, y, text, fg, bg):
    For each char: font_draw_char, advance x by font->width.
    Return total width drawn.
  font_text_width(font, text) → pixels.
  font_text_height(font) → pixels.
  Built-in fonts: embed at least 2 PSF2 fonts in the binary:
    - 8x16 monospace (terminal/code font, ~4KB).
    - 8x14 or 6x12 small font (UI labels, ~3KB).
  Anti-aliased rendering (stretch goal):
    Store 8-bit alpha per pixel instead of 1-bit.
    Blend: out = (fg * alpha + bg * (255 - alpha)) / 255.

PHASE 23 - SIGNALS, PIPES, AND THREADS
=====================================================================
These are the fundamental IPC and process control primitives
needed for a real multi-process OS.

kernel/proc/signal.c:
  Signal delivery mechanism (subset of POSIX signals):
  Signal numbers:
    SIGTERM (15): request termination.
    SIGKILL (9):  force kill (cannot be caught).
    SIGINT  (2):  interrupt (Ctrl+C).
    SIGCHLD (17): child process terminated.
    SIGSEGV (11): segmentation fault.
    SIGFPE  (8):  arithmetic error (div by zero).
    SIGALRM (14): alarm timer expired.
    SIGPIPE (13): write to broken pipe.
    SIGUSR1 (10): user-defined signal 1.
    SIGUSR2 (12): user-defined signal 2.
    SIGSTOP (19): stop process (cannot be caught).
    SIGCONT (18): continue stopped process.

  struct fry_process extensions:
    uint64_t sig_pending;          // bitmask of pending signals
    uint64_t sig_blocked;          // bitmask of blocked signals
    void (*sig_handlers[32])(int); // user-space handler function pointers
                                   // SIG_DFL (0) = default action
                                   // SIG_IGN (1) = ignore

  Default actions:
    SIGTERM, SIGINT, SIGUSR1/2, SIGPIPE, SIGALRM: terminate process.
    SIGKILL: terminate immediately (no handler, no block).
    SIGSTOP: stop process (set state = STOPPED).
    SIGCONT: resume stopped process.
    SIGCHLD: ignore (unless handler installed).
    SIGSEGV, SIGFPE: terminate + core dump (or just terminate).

  signal_send(pid, signum):
    Find process by pid. Set bit in sig_pending.
    If process state == SLEEPING: wake it up.
    If SIGKILL: immediately mark process ZOMBIE, run cleanup.
    If SIGSTOP: set state = STOPPED, remove from run queue.
    If SIGCONT: set state = RUNNING, add to run queue.

  signal_check(proc):
    Called on return to userspace (in syscall_return or iret path).
    Scan sig_pending & ~sig_blocked for set bits.
    For first pending signal:
      Clear bit from sig_pending.
      If handler == SIG_DFL: execute default action.
      If handler == SIG_IGN: discard.
      If handler == user function:
        Save current user registers (rip, rsp, rflags) on user stack.
        Push signal number as argument.
        Set user rip = handler address.
        Set user rsp = adjusted stack with return trampoline.
        On return from handler: sigreturn syscall restores saved regs.

  kernel/proc/syscall.c signal syscalls:
    SYS_KILL(60): pid, signum → send signal.
    SYS_SIGNAL(61): signum, handler_ptr → install handler. Returns old handler.
    SYS_SIGRETURN(62): restore saved context after signal handler.
    SYS_SIGPROCMASK(63): how, set_ptr, oldset_ptr → block/unblock signals.
    SYS_ALARM(64): seconds → set SIGALRM timer.
    SYS_PAUSE(65): suspend until signal delivered.
    SYS_WAITPID(66): pid, status_ptr, options → wait for child.

  user/libc/libc.c signal wrappers:
    int fry_kill(int pid, int sig);
    void *fry_signal(int sig, void (*handler)(int));
    int fry_sigprocmask(int how, const uint64_t *set, uint64_t *oldset);
    unsigned int fry_alarm(unsigned int seconds);
    int fry_pause(void);
    int fry_waitpid(int pid, int *status, int options);

  Ctrl+C handling:
    When GUI compositor receives Ctrl+C keypress:
      If focused window has a process: signal_send(pid, SIGINT).
    Shell default SIGINT handler: cancel current input line.
    Apps: default terminates unless handler installed.

kernel/proc/pipe.c:
  In-kernel pipe implementation:
  struct pipe {
    uint8_t  buf[PIPE_BUF_SIZE];  // 4096 bytes circular buffer
    uint32_t read_pos;
    uint32_t write_pos;
    uint32_t count;               // bytes in buffer
    int      read_fd_open;        // 0 if read end closed
    int      write_fd_open;       // 0 if write end closed
    uint32_t reader_pid;          // process waiting to read (0 if none)
    uint32_t writer_pid;          // process waiting to write (0 if none)
  };
  #define MAX_PIPES 64
  static struct pipe pipes[MAX_PIPES];

  pipe_create(read_fd_out, write_fd_out):
    Allocate pipe. Create two fd entries in process fd table.
    read_fd: reads from pipe. write_fd: writes to pipe.
  pipe_read(pipe, buf, len):
    If count == 0 and write_fd_open: block (fry_sleep loop until data).
    If count == 0 and !write_fd_open: return 0 (EOF).
    Copy min(len, count) bytes from buf at read_pos. Advance read_pos.
    Wake writer if was blocked.
  pipe_write(pipe, buf, len):
    If !read_fd_open: send SIGPIPE to caller, return -1.
    If buffer full: block until space (fry_sleep loop).
    Copy min(len, space) bytes to buf at write_pos. Advance write_pos.
    Wake reader if was blocked.
  pipe_close_read(pipe): set read_fd_open = 0. Wake blocked writer.
  pipe_close_write(pipe): set write_fd_open = 0. Wake blocked reader.

  SYS_PIPE(67): pipe_fds_ptr → creates pipe, writes [read_fd, write_fd].
  SYS_DUP2(68): old_fd, new_fd → duplicate fd (for redirection).

  user/libc/libc.c pipe wrappers:
    int fry_pipe(int fds[2]);
    int fry_dup2(int oldfd, int newfd);

  Shell redirection (shell.c extensions):
    Parse "|": spawn left command with stdout→pipe_write,
               spawn right command with stdin→pipe_read.
    Parse ">": redirect stdout to file (open file, dup2 to fd 1).
    Parse "<": redirect stdin from file (open file, dup2 to fd 0).
    Parse ">>": redirect stdout to file in append mode.
    Multiple pipes: cmd1 | cmd2 | cmd3 → chain of pipes.

kernel/proc/thread.c:
  Kernel-level threading support:
  struct fry_thread {
    uint32_t tid;
    uint32_t pid;            // parent process
    uint64_t saved_rsp;
    uint64_t saved_rip;
    uint64_t kernel_stack_top;
    uint8_t  state;          // RUNNING, SLEEPING, ZOMBIE
    struct fry_thread *next; // scheduler thread queue
  };
  Each process can have up to 16 threads.
  All threads in a process share: address space (cr3), fd table, signal handlers.
  Each thread has its own: stack, register set, TLS pointer.

  thread_create(entry_func, arg, stack_size) → tid:
    Allocate fry_thread. Allocate user stack (stack_size or 64KB default).
    Set saved_rip = entry_func, saved_rsp = stack_top.
    Push arg as first argument (rdi register).
    Allocate kernel stack (16KB).
    Add to parent process thread list and scheduler run queue.
  thread_exit(exit_code):
    Mark thread ZOMBIE. Free kernel stack.
    If last thread in process: process exit.
    Signal parent thread if joinable.
  thread_join(tid) → exit_code:
    Block until target thread exits. Return exit code. Free thread.

  Scheduler changes (sched.c):
    Run queue now contains threads, not just processes.
    sched_tick: round-robin through all RUNNING threads.
    context_switch: save/restore per-thread register state.
    Same process threads: no CR3 switch needed (optimization).

  SYS_THREAD_CREATE(70): entry, arg, stack_size → tid.
  SYS_THREAD_EXIT(71): exit_code → no return.
  SYS_THREAD_JOIN(72): tid → exit_code.

  Synchronization primitives:
    struct fry_mutex {
      volatile uint32_t locked;  // 0=unlocked, 1=locked
      uint32_t owner_tid;
      uint32_t waiters[16];      // TIDs waiting
      uint8_t  waiter_count;
    };
    mutex_lock(m): atomic cmpxchg. If contended: add to waiters, sleep.
    mutex_unlock(m): set locked=0. Wake first waiter.
    SYS_MUTEX_LOCK(73), SYS_MUTEX_UNLOCK(74), SYS_MUTEX_INIT(75).

  user/libc/libc.c thread wrappers:
    int fry_thread_create(void (*entry)(void*), void *arg, long stack_size);
    void fry_thread_exit(int code);
    int fry_thread_join(int tid);
    int fry_mutex_init(fry_mutex_t *m);
    int fry_mutex_lock(fry_mutex_t *m);
    int fry_mutex_unlock(fry_mutex_t *m);


PHASE 24 - USB MASS STORAGE AND EXTENDED USB
=====================================================================
Phase 10 provides xHCI + hub + HID. This phase adds USB storage
so external drives and USB sticks work as filesystems.

kernel/drivers/usb/usb_mass_storage.c:
  USB class driver: class=0x08 (Mass Storage), subclass=0x06 (SCSI),
  protocol=0x50 (Bulk-Only Transport, BOT).

  BOT protocol (USB Mass Storage spec):
    Command Block Wrapper (CBW, 31 bytes):
      dCBWSignature: 0x43425355 ("USBC").
      dCBWTag: unique tag per command (incrementing counter).
      dCBWDataTransferLength: expected data bytes.
      bmCBWFlags: 0x80 = data-in (device→host), 0x00 = data-out.
      bCBWLUN: logical unit number (usually 0).
      bCBWCBLength: length of SCSI command (6-16 bytes).
      CBWCB[16]: SCSI command block.
    Command Status Wrapper (CSW, 13 bytes):
      dCSWSignature: 0x53425355 ("USBS").
      dCSWTag: must match CBW tag.
      dCSWDataResidue: bytes not transferred.
      bCSWStatus: 0x00=passed, 0x01=failed, 0x02=phase error.

  usb_msc_probe(usb_device):
    Find Bulk-IN and Bulk-OUT endpoints from interface descriptor.
    Send GET_MAX_LUN request (class-specific, returns max LUN index).
    For each LUN (typically just LUN 0):
      Send INQUIRY command (SCSI opcode 0x12):
        CBW with CBWCB = {0x12, 0, 0, 0, 36, 0}.
        Read 36-byte inquiry response: device type, vendor, product.
        Device type 0x00 = direct access (disk).
      Send TEST_UNIT_READY (0x00): verify device is ready.
        May need retry with REQUEST_SENSE (0x03) if not ready.
      Send READ_CAPACITY(10) (0x25):
        Response: last_lba (4 bytes, big-endian) + block_size (4 bytes).
        Total capacity = (last_lba + 1) * block_size.
      Register as block device with VFS:
        name "usb0" (or usbN for multiple devices).
        block_size, total_blocks.
        read function → usb_msc_read.
        write function → usb_msc_write.

  usb_msc_read(lun, lba, buf, sector_count):
    Build READ(10) SCSI command (opcode 0x28):
      CBWCB = {0x28, 0, lba[3], lba[2], lba[1], lba[0],
               0, count[1], count[0], 0}.
    Build CBW: data_transfer_length = sector_count * block_size, flags=0x80.
    Send CBW via Bulk-OUT endpoint (xhci_submit_bulk).
    Read data via Bulk-IN endpoint (sector_count * block_size bytes).
    Read CSW via Bulk-IN endpoint (13 bytes).
    Verify CSW tag matches, status == 0.
    If status == 1: issue REQUEST_SENSE, log error, return -1.

  usb_msc_write(lun, lba, buf, sector_count):
    Build WRITE(10) command (opcode 0x2A):
      Same format as READ(10) but flags=0x00 (data-out).
    Send CBW via Bulk-OUT.
    Send data via Bulk-OUT.
    Read CSW via Bulk-IN. Verify status.

  Hot-plug support:
    xHCI Port Status Change event → detect device disconnect.
    On disconnect: unmount filesystem, deregister block device.
    On connect: re-enumerate, probe, mount if FAT32 detected.

  Auto-mount:
    When USB mass storage registered: read sector 0 (MBR or GPT).
    If GPT: parse partition table.
    If FAT32 signature found: mount on /usb0 (or next available).
    Notify GUI (compositor) of new mount for file manager.

kernel/drivers/usb/usb_audio.c (stretch goal):
  USB class driver: class=0x01 (Audio).
  Basic USB audio playback to USB headsets/speakers.
  Parse Audio Control and Audio Streaming interface descriptors.
  Set sample rate via SET_CUR request.
  Stream PCM data via isochronous endpoint.
  Register as audio output device (alternative to HDA).


PHASE 25 - ADVANCED GRAPHICS
=====================================================================
Move from software-only rendering to hardware-assisted graphics.

kernel/drivers/video/intel_gpu.c:
  Intel Gen9 (Coffee Lake GT2) basic modesetting.
  PCI device: vendor=0x8086, class=0x03 (Display Controller).
  Dell Precision 7530 has Intel UHD Graphics 630 (device ID 0x3E92 or similar).

  GPU register access:
    BAR0 (MMIO Graphics Registers, ~16MB):
      PIPE_A registers at 0x70000+.
      PLANE registers at 0x70180+.
      DPLL (Display PLL) at 0x6014+.
      GMBUS (I2C for EDID) at 0x5100+.
    BAR2 (GTT aperture, ~256MB): maps GPU-visible memory.

  intel_gpu_probe(pci_dev):
    Map BAR0. Read PCI device ID to determine Gen version.
    Read current display mode from GOP (already set by UEFI firmware).
    DO NOT reprogram display pipes on initial boot — UEFI already set
    the mode. Just take over the framebuffer address.
    Read DSPCNTR (Display/Sprite Control) to find active planes.
    Read DSPSURF (Display Surface Base Address) for current FB.
    Map stolen memory (where UEFI placed the framebuffer).

  Hardware cursor:
    CUR_CTL (Cursor Control register): enable, format, size.
    CUR_BASE (Cursor Base Address): physical address of cursor image.
    CUR_POS (Cursor Position): x[15:0] | y[31:16].
    Cursor format: 64x64 ARGB8888 (16KB).
    Upload cursor image to GPU-visible memory.
    Set CUR_BASE to cursor image address.
    On mouse move: write CUR_POS only (no software redraw).
    Eliminates cursor tearing and reduces CPU load.

  Page flipping (vsync-aligned buffer swap):
    Allocate two framebuffers (front and back).
    Compositor renders to back buffer.
    On vsync (PIPE_A VBLANK interrupt): swap DSPSURF to back buffer.
    This eliminates all tearing.
    VBLANK interrupt: enable via PIPE_A interrupt control register.
    Vector: shared with other display interrupts (DE_PIPE_A).
    ISR: check PIPE_A_VBLANK bit, signal compositor to swap.

  Stretch goals (not required for daily driver):
    2D blitter acceleration (BLT ring): hardware fill, copy.
    Render command streamer: GPU-accelerated compositing.
    GTT (Graphics Translation Table) management for VRAM.

src/user/gui/gui.c compositor enhancements:
  Damage tracking (dirty rectangles):
    struct dirty_rect { int x, y, w, h; };
    Track which screen regions changed each frame:
      Window moved: old position + new position are dirty.
      Window content updated (TW_MSG_UPDATE): window rect is dirty.
      Cursor moved: old cursor rect + new cursor rect dirty.
    On redraw: only repaint dirty regions, not entire screen.
    Reduces CPU usage significantly when most of screen is static.

  Alpha blending / transparency:
    Window shadows: draw semi-transparent dark rect behind each window.
    Compositor blend: for each pixel in overlap:
      out.r = (src.r * src.a + dst.r * (255 - src.a)) / 255.
      (same for g, b).
    Enables: translucent window borders, fade effects, tooltips.

  Z-order management:
    Linked list of windows sorted by Z.
    Click window: move to top of Z stack.
    Alt+Tab: cycle through windows (bring next to top).
    Minimized windows: skip in draw order, show in taskbar.

  Taskbar:
    Bottom of screen, 32px tall, full width.
    Left: "Start" button (app launcher menu).
    Center: running app buttons (click to focus/minimize).
    Right: clock (HH:MM from kernel time), volume icon, network icon.
    App launcher menu: list of available .fry apps on disk.
    Click app name: spawn process.


PHASE 26 - WIFI WPA2 AND FULL NETWORKING
=====================================================================
Phase 12 provides the Intel 9260 driver with raw 802.11 frame I/O.
Phase 19 provides TCP/IP. This phase bridges them with WiFi
authentication and practical network tools.

kernel/drivers/net/wifi_auth.c:
  WPA2-Personal (PSK) authentication:
  IEEE 802.11i / WPA2 handshake:
    1. Association:
       Send Authentication frame (Open System, seq=1).
       Receive Authentication frame (seq=2, status=0).
       Send Association Request: SSID, supported rates, RSN IE.
       Receive Association Response: AID, status=0.
    2. 4-Way Handshake (EAPOL-Key):
       Key derivation:
         PMK = PBKDF2(passphrase, SSID, 4096, 32) — SHA1-based.
           PBKDF2: iterate HMAC-SHA1 4096 times.
           HMAC-SHA1: standard keyed hash (RFC 2104).
         PTK = PRF-384(PMK, "Pairwise key expansion",
                       min(AA,SPA) || max(AA,SPA) ||
                       min(ANonce,SNonce) || max(ANonce,SNonce)).
           AA = AP MAC, SPA = station MAC.
           PRF: HMAC-SHA1 iterated to produce desired length.
         KCK = PTK[0:15]   (Key Confirmation Key, 16 bytes).
         KEK = PTK[16:31]  (Key Encryption Key, 16 bytes).
         TK  = PTK[32:47]  (Temporal Key, 16 bytes, for CCMP).
       Message 1 (AP→STA): EAPOL-Key with ANonce.
       Message 2 (STA→AP): EAPOL-Key with SNonce, MIC (HMAC-SHA1 with KCK).
       Message 3 (AP→STA): EAPOL-Key with GTK (encrypted with KEK), MIC.
         Decrypt GTK using AES key unwrap (RFC 3394) with KEK.
       Message 4 (STA→AP): EAPOL-Key ACK, MIC.
    3. After handshake: install TK for unicast, GTK for multicast.

  Crypto primitives (kernel/crypto/):
    sha1.c: SHA-1 hash (for HMAC, PBKDF2).
    hmac_sha1.c: HMAC-SHA1 (RFC 2104).
    pbkdf2.c: PBKDF2-SHA1 (RFC 2898).
    aes.c: AES-128 encrypt/decrypt (for CCMP, key unwrap).
    ccmp.c: AES-CCMP encrypt/decrypt (802.11i per-frame encryption).
      CCMP (Counter mode with CBC-MAC):
        Nonce = priority | A2 (source MAC) | packet number (6 bytes).
        AAD = FC | A1 | A2 | A3 | SC (from 802.11 header).
        Encrypt: AES-CTR with nonce. Authenticate: AES-CBC-MAC.
        8-byte MIC appended to frame.
    aes_keywrap.c: AES Key Wrap (RFC 3394) for GTK decryption.

  wifi_connect(ssid, passphrase):
    Scan for AP: send Probe Request, wait for Probe Response matching SSID.
    Associate with AP (Authentication + Association frames).
    Derive PMK via PBKDF2.
    Execute 4-way handshake.
    Install keys. Mark connection established.
    Start DHCP (Phase 19) to get IP address.

  wifi_send_data(buf, len):
    Encrypt with CCMP using TK.
    Build 802.11 data frame (QoS if supported).
    Send via wifi_9260 driver's TX path.

  wifi_receive_data(buf, len):
    Strip 802.11 header.
    Decrypt CCMP using TK (or GTK for multicast).
    Pass decrypted payload to net_receive() as Ethernet frame.

src/user/apps/netmgr.c - Network Manager:
  GUI app for network configuration.
  Features:
    - WiFi: scan available networks, show signal strength (RSSI).
    - Connect: select network, enter passphrase, connect.
    - Status: connected/disconnected, IP address, signal, speed.
    - Ethernet: show link status, IP configuration.
    - Saved networks: store SSID + passphrase (in config file on disk).
    - Static IP: manual IP/subnet/gateway/DNS entry.
  TaterWin window with widget library (Phase 21).

src/user/apps/wget.c - HTTP downloader:
  Minimal HTTP/1.1 client:
    DNS resolve hostname. TCP connect to port 80.
    Send: "GET /path HTTP/1.1\r\nHost: hostname\r\n\r\n".
    Parse response: status line, headers (Content-Length), body.

PHASE 27 - MULTI-USER AND SECURITY
=====================================================================
Process isolation, user accounts, and access control.

kernel/security/users.c:
  User account management:
  struct user_account {
    uint32_t uid;
    uint32_t gid;
    char     username[32];
    uint8_t  password_hash[32];  // SHA-256 of password
    char     home_dir[64];       // e.g., "/home/admin"
    char     shell[32];          // e.g., "/SHELL.FRY"
    uint8_t  flags;              // ADMIN, LOCKED, etc.
  };
  User database: stored as flat file /etc/users.dat on disk.
  At boot: load user database into memory.
  user_authenticate(username, password) → uid or -1:
    Hash password with SHA-256. Compare against stored hash.
  user_create(username, password, is_admin) → uid:
    Allocate UID (starting from 1000 for regular users).
    Create home directory.
    Write to users.dat.
  Root user (UID 0): always exists, full access to everything.
  Default user "admin" (UID 1000): created on first boot.

kernel/security/perms.c:
  File permission model (simplified Unix-style):
  Per-file metadata in FAT32: use reserved bytes in directory entry
  (bytes 12-13, which FAT32 spec reserves as "NTRes" and "CrtTimeTenth")
  to store owner UID (8 bits) and permission flags (8 bits).
  Permission bits: owner_rwx (3 bits), other_rwx (3 bits).
    Read (r): can read file contents / list directory.
    Write (w): can modify file / create/delete entries in directory.
    Execute (x): can execute file / enter directory.
  perm_check(uid, file_owner, file_perms, requested_access) → allow/deny:
    If uid == 0 (root): always allow.
    If uid == file_owner: check owner bits.
    Else: check other bits.

kernel/proc/process.c extensions:
  struct fry_process gains:
    uint32_t uid;      // effective user ID
    uint32_t gid;      // effective group ID
    uint32_t euid;     // effective UID (for setuid)
  Process inherits UID from parent on fork/spawn.
  Syscalls check UID permissions before allowing:
    File operations: perm_check before open/create/delete.
    Signal operations: can only signal own processes (or root).
    Process operations: can only read own process info (or root).

  SYS_SETUID(80): uid → change effective UID (root only).
  SYS_GETUID(81): → return current UID.
  SYS_SETGID(82): gid → change effective GID.
  SYS_GETGID(83): → return current GID.

src/user/apps/login.c - Login Screen:
  First app launched by GUI compositor on boot.
  Username field + password field (masked with *).
  On submit: SYS_AUTH(84) syscall → user_authenticate().
  On success: set process UID to authenticated user.
    Launch shell or desktop session for that user.
  On failure: show "Invalid credentials", clear password field.
  No bypass: login screen is the only way to enter the desktop.

src/user/apps/su.c - Switch User:
  Command-line tool to switch to another user (like sudo).
  Usage: su <username>
  Prompts for password. On success: spawn shell with new UID.
  If current UID == 0: no password required (root can su to anyone).


PHASE 28 - DYNAMIC LINKING AND PACKAGE MANAGEMENT
=====================================================================
Shared libraries reduce binary size and enable code reuse.
Package management enables installing new software.

kernel/proc/dynlink.c and user/libc/dynlink.c:
  Shared library format: .frylib (same as .fry but with relocations).
  ELF64 shared object support:
    ELF header: e_type = ET_DYN (3).
    Program headers: PT_LOAD segments (position-independent).
    Dynamic section (PT_DYNAMIC): DT_NEEDED, DT_SYMTAB, DT_STRTAB,
      DT_RELA, DT_RELASZ, DT_HASH, DT_SONAME.
    Relocation types: R_X86_64_RELATIVE, R_X86_64_64,
      R_X86_64_GLOB_DAT, R_X86_64_JUMP_SLOT.
  Dynamic linker (ld-fry.so equivalent):
    Embedded in libc or loaded as separate .frylib at fixed address.
    On process start (before main()):
      Parse ELF dynamic section of executable.
      For each DT_NEEDED: load .frylib from /lib/ directory.
        Map .frylib PT_LOAD segments into process address space.
        Apply relocations: resolve symbols from loaded libraries.
      Build GOT (Global Offset Table) and PLT (Procedure Linkage Table).
      Lazy binding (optional): PLT stubs call linker on first use.
    Runtime loading (dlopen-like):
      fry_dlopen(path) → handle: load .frylib at runtime.
      fry_dlsym(handle, symbol_name) → address.
      fry_dlclose(handle): unmap library.
  SYS_MMAP(85): map memory region (needed for loading libraries).
  SYS_MUNMAP(86): unmap memory region.
  Shared libc: libc.frylib containing all libc functions.
    Apps link against libc.frylib instead of static libc.a.
    Saves ~20-50KB per binary.

Package management:
  Package format (.frypkg):
    tar archive containing:
      manifest.txt: name, version, description, dependencies.
      bin/: executable .fry files → install to /usr/bin/.
      lib/: shared .frylib files → install to /lib/.
      etc/: configuration files.
      data/: app-specific data files.
  pkg_install(path):
    Read manifest. Check dependencies (are they installed?).
    Extract files to target directories.
    Register in /etc/packages.dat (installed package database).
  pkg_remove(name):
    Look up package in database. Remove installed files.
    Check reverse dependencies (warn if other packages need it).
  pkg_list(): show all installed packages with versions.

  src/user/apps/pkgmgr.c - Package Manager:
    CLI tool for package operations:
      pkgmgr install <file.frypkg>
      pkgmgr remove <name>
      pkgmgr list
      pkgmgr info <name>
    Future: repository support via HTTP (fetch package index,
    download .frypkg files, resolve dependencies automatically).
    Requires: Phase 18 (writable FS), Phase 19 (TCP/IP for remote repos).
